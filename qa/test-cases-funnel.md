# Тест-кейсы: Отчёт «Воронка продаж»

Дата: 2026-03-02
Правило каталога: один тест-кейс проверяет одну причину и один основной ожидаемый результат.

## TC-01. График строится сразу при открытии

Приоритет: Critical

Предусловия:

- выбран сценарий `smoke-open-report`

Шаги:

1. Открыть отчёт.
2. Дождаться окончания загрузки.

Ожидаемый результат:

- отчёт открывается сразу на графике;
- на графике есть 5 этапов и непустые данные;
- KPI и блок сотрудников тоже заполнены;
- alert-баннеры не мешают сценарию, потому что здесь их нет.

## TC-02. Count-fallback без суммы

Приоритет: Critical

Предусловия:

- выбран сценарий `metric-count-no-amount`

Шаги:

1. Открыть отчёт.
2. Проверить график, шапку блока и alert.
3. Открыть `Настроить`.

Ожидаемый результат:

- график построен по количеству сделок;
- суммы на графике не показываются;
- в шапке графика нет кнопок переключения метрики;
- в модалке настройки нет блока выбора числового поля суммы;
- показан только alert `NO_AMOUNT_FIELD`;
- в alert есть ссылка `Как добавить сумму`, ведущая на статью Kaiten FAQ.

## TC-03. Amount по умолчанию с общим полем суммы

Приоритет: Critical

Предусловия:

- выбран сценарий `metric-amount-common-field`

Шаги:

1. Открыть отчёт.
2. Проверить график и KPI сразу после загрузки.

Ожидаемый результат:

- в шапке графика нет кнопок переключения метрики;
- график сразу строится по денежным значениям;
- ось и бейджи графика показывают суммы;
- KPI сразу отображаются в денежной логике, без ручного переключения.
