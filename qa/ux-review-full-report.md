# UI/UX обзор: Отчёт «Воронка продаж»

**Дата:** 2026-03-02
**Scope:** Полный UI/UX обзор фронтенд-прототипа
**Метод:** Визуальный осмотр + анализ исходного кода + a11y-анализ

---

## 1. Первый заход (First-Run Experience)

### Положительные стороны
- Воронка строится сразу при первом открытии — нет стоп-экрана (P-001 из CPO-фидбека решён)
- Alert-баннеры информируют о неоднозначности автоконфигурации
- Кнопка «Настроить» на каждом алерте позволяет перейти к коррекции

### Проблемы

**UX-001. Два алерта одновременно перегружают первый экран**
На первом заходе пользователь видит 2 warning-алерта, которые занимают ~30% видимой области. Это создаёт ощущение, что с отчётом что-то не так, хотя воронка работает корректно.

Рекомендация: объединить алерты в один компактный баннер или показывать по одному.

Скриншот: `screenshots/01-default-view-chart-count.png`

**UX-002. Алерт не объясняет, ЧТО именно было определено автоматически**
Текст «Последняя определена как Выигран, остальные — как Проигран» не называет конкретные колонки. Пользователь не знает, какую из 4 done-колонок система выбрала как «Выигран» (302 = «Активный клиент»).

Рекомендация: включить названия колонок: «"Активный клиент" определена как "Выигран", "Квалифицирован", "Оплачено", "Отказ" — как "Проигран".»

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/lib/best-guess.ts`, строки 44-49

## 2. Информационная архитектура

### Положительные стороны
- Breadcrumbs «Отчёты / Воронка продаж» дают контекст
- KPI-карточки под графиком дают агрегированные числа
- Секция «По сотрудникам» даёт разбивку по ответственным

### Проблемы

**UX-003. Три уровня вложенности данных без навигации**
Страница содержит: (1) chart/table + KPI + employees — всё на одной scrollable странице. Нет якорей, табов или навигации. На длинных списках сотрудников пользователю приходится скроллить далеко вниз.

Рекомендация: добавить якорные ссылки или sticky-заголовки секций.

**UX-004. Заголовок «Конверсия по этапам: Сделки» не информативен**
Не отражает:
- Какое пространство выбрано
- Какой период показан
- Какой режим метрики активен

Рекомендация: «Воронка продаж · CRM-пространство · Текущий месяц · По количеству»

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/funnel-report.tsx`, строка 183

**UX-005. Переключатели MetricMode и ViewMode визуально равнозначны**
Оба выглядят одинаково (MUI ToggleButtonGroup). Пользователю может быть непонятно, что «Кол-во / Сумма» влияет на данные, а «График / Таблица» — только на представление.

Рекомендация: визуально разделить — например, метрику вынести на уровень фильтров.

Скриншот: `screenshots/01-default-view-chart-count.png`

## 3. Системная обратная связь (System Status & Feedback)

### Проблемы

**UX-006. Нет подтверждения после применения настроек**
Пользователь нажимает «Применить» в диалоге настроек — диалог закрывается, но нет toast/snackbar, подтверждающего, что настройки применены. Алерты не обновляются.

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/funnel-report.tsx`, строки 90-95

**UX-007. Нет индикации загрузки при применении фильтров**
Кнопка «Показать» не показывает loading-состояние. При реальном API пользователь не будет знать, что запрос в процессе.

**UX-008. Dismissed алерты не восстанавливаются**
Если пользователь скрывает алерт, он не может его вернуть без перезагрузки страницы. Нет кнопки «Показать скрытые уведомления».

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/funnel-report.tsx`, строки 97-107

## 4. Формы и фильтры

### Проблемы

**UX-009. Фильтр «Период» и даты дублируют друг друга**
Select «Период» (предустановленные) и поля «Дата начала» / «Дата окончания» показаны одновременно. Неясно:
- Если выбрал «Текущий месяц», можно ли менять даты?
- Если изменил дату вручную, сбрасывается ли Select?

Текущая реализация: Select обновляет даты, но ручное изменение дат не обновляет Select. Это несогласованность.

Файлы:
- `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/filters-sidebar.tsx`, строки 85-154
- `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/funnel-report.tsx`, строки 35-36

**UX-010. Нет валидации дат**
Можно установить дату начала позже даты окончания. Нет визуальной индикации ошибки.

**UX-011. Placeholder «Все» в фильтре ответственных**
При пустом выборе показывается placeholder «Все», что корректно. Но после выбора и сброса менеджеров placeholder не возвращается — поле остаётся пустым.

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/filters-sidebar.tsx`, строка 192

## 5. Таблицы, графики и метрики

### Положительные стороны
- Conversion badges с цветовой индикацией (зелёный/жёлтый/красный)
- Tooltip на этапах в таблице показывает название доски
- Custom tooltip на графике показывает все метрики

### Проблемы

**UX-012. График: CR% на баре — это conversion_to_win, а в конверсионных стрелках — conversion_to_next**
Два разных типа конверсии показаны рядом без объяснения:
- **CR 20%** внутри бара — доля выигранных от общего числа вошедших (conversion_to_win)
- **82.0% ->** под баром — доля перешедших на следующий этап (conversion_to_next)

Пользователь может спутать эти метрики. Нет легенды или пояснения.

Скриншот: `screenshots/01-default-view-chart-count.png`

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/funnel-chart.tsx`, строки 114-116 vs 329-336

**UX-013. Кнопка «Сумма» неактивна без объяснения на экране**
Переключатель «Сумма» — disabled. Tooltip появляется при наведении: «Нет числового поля для суммы. Настройте в параметрах.» Но на тач-устройствах hover не работает.

Скриншот: `screenshots/01-default-view-chart-count.png`

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/metric-mode-toggle.tsx`, строки 42-48

**UX-014. KPI-карточки не объясняют источник чисел**
- «100 сделок всего» — это total_entered (вошедших в первый этап), а не общее количество карточек
- «20 оплачено» — это total_won, хотя колонка «Оплачено» в BestGuess помечена как «Проиграно»
- Нет подписи «за период» или «по всем доскам»

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/kpi-cards.tsx`

**UX-015. Drilldown: все сделки — одного месяца (июнь 2024)**
Mock-данные нереалистичны — все 100 сделок за один месяц. При реальных данных будет разброс. Но для прототипа это приемлемо.

## 6. Accessibility и взаимодействие

### Проблемы

**UX-016. Бары графика не доступны с клавиатуры**
Recharts `<Bar>` элементы кликабельны мышью, но не фокусируемы с клавиатуры. Пользователь не может открыть drilldown без мыши.

**UX-017. Кнопка «Подробнее:» без aria-label и без фокусного кольца**
Элемент `<Box component="button">` без `aria-label`, без `type="button"`, без видимого `outline` при фокусе.

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/employee-section.tsx`, строки 96-118

**UX-018. Цветовая индикация без текстовых маркеров**
- Зелёный KPI = «оплачено», красный = «потеряно» — различаются только цветом. Но в карточках есть текст, так что частично решено.
- Conversion badges (зелёный / жёлтый / красный) в таблице различаются только цветом. Нет иконок или текстовых маркеров для дальтоников.

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/conversion-badge.tsx`

**UX-019. Console warnings об aria-hidden + focus при Drawer**
При закрытии drilldown-drawer возникает accessibility warning. Это баг MUI, но его нужно обработать.

## 7. Responsive-поведение

### Проблемы

**UX-020. Sidebar фильтров фиксированной ширины не адаптируется**
На экранах < 1024px sidebar (280px) + основной контент не помещаются. Нет responsive breakpoints: sidebar не сворачивается и не превращается в drawer.

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/filters-sidebar.tsx`, строки 39-49

**UX-021. Названия этапов на графике обрезаются при 6+ этапах**
При 6 этапах и ширине < 1200px названия «Квалификация», «Предложение», «Переговоры» обрезаются через `text-overflow: ellipsis`. Нет tooltip на обрезанных названиях.

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/funnel-chart.tsx`, строки 246-259

**UX-022. Drilldown drawer: xs=100%, sm=540px**
На мобильных (xs) drawer занимает 100% ширины — корректно. На sm и выше — 540px. Но sidebar фильтров (280px) не скрывается при открытии drawer, создавая наложение.

Файл: `/Users/slvn/ai/kaiten/funnel_report/app/components/funnel/stage-drilldown.tsx`, строка 53

## 8. Типичные упущения дизайна

| Упущение | Статус | Комментарий |
|----------|--------|-------------|
| Нет состояния «нет данных» | Отсутствует | Если фильтры отсекут все данные |
| Нет состояния «частично сконфигурировано» | Частично есть | Alert-баннеры покрывают |
| Нет состояния «нет прав» | Отсутствует | Не применимо для прототипа |
| Нет предупреждения о необратимом действии | Частично | Сброс к автоматическим не требует подтверждения |
| Нет объяснения auto-defaults | Есть | Alert-баннеры объясняют |
| Нет объяснения пустых данных | Отсутствует | «—» без контекста |
| Нет различия saved/unsaved | Отсутствует | Setup dialog: cancel закрывает без предупреждения |
