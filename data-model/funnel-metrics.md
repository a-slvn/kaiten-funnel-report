# Каталог метрик: Отчёт «Воронка продаж»

## Общее описание

Данный документ описывает полный набор метрик отчёта «Воронка продаж» в Kaiten CRM. Метрики разделены на три группы:

1. **Метрики этапа** -- характеризуют каждый этап воронки в отдельности
2. **Метрики всей воронки** -- агрегированные показатели по всей воронке
3. **Метрики сравнения периодов** -- дельты и тренды для анализа динамики

Для каждой метрики указаны: определение, точная формула, единица измерения, формат отображения, edge-кейсы и бенчмарк.

---

## Входные параметры расчёта

Все метрики зависят от следующих параметров, задаваемых пользователем при построении отчёта:

| Параметр | Тип | Описание |
|----------|-----|----------|
| `board_id` | integer | Доска, по которой строится воронка |
| `stage_order` | integer[] | Упорядоченный массив `column_id`, определяющий этапы воронки слева направо |
| `win_column_ids` | integer[] | Колонки, означающие победу (Выигран) |
| `loss_column_ids` | integer[] | Колонки, означающие проигрыш (Проигран) |
| `date_from` | date | Начало периода (включительно) |
| `date_to` | date | Конец периода (включительно) |
| `filters` | object | Дополнительные фильтры: ответственный, источник, теги, диапазон суммы |

---

## Принцип фильтрации по периоду (когортный подход)

Все поточные метрики рассчитываются по **когорте** — набору сделок, привязанных к периоду по одной дате-якорю.

**Дата-якорь сделки** = дата первого входа на любой этап воронки: `MIN(entered_at)` по всем записям в `funnel_deal_history`.

**Когорта периода** = все сделки, чья дата-якорь попадает в `[date_from, date_to]`.

Для сделок из когорты показываются **все** этапы, которые они прошли — независимо от дат перемещений. Это гарантирует:
- Каждая сделка принадлежит **ровно одному** периоду
- Числа на этапах воронки убывают сверху вниз (каждый следующий этап <= предыдущего)
- Drill-down по любому числу показывает конкретный список карточек
- Данные сходятся с тем, что видно на доске

**Snapshot-метрики** (pipeline, зависшие) — показывают состояние на `date_to`, независимо от когорты.

| Тип метрики | Как фильтруется | Примеры |
|-------------|-----------------|---------|
| Поточная (когортная) | По дате первого входа в воронку | Кол-во, конверсия, время, drop-off, потери, velocity |
| Snapshot | По состоянию на `date_to` | Pipeline value, взвешенный pipeline, зависшие |

## 1. Метрики этапа

Рассчитываются для каждого этапа воронки (каждого элемента `stage_order`).

### 1.1. Количество сделок, вошедших на этап

| Поле | Значение |
|------|----------|
| **Название** | Количество сделок на этапе (Deals entered stage) |
| **Определение** | Число уникальных сделок из когорты периода, которые побывали на данном этапе |
| **Формула** | `COUNT(DISTINCT card_id) WHERE card_id IN cohort AND stage = S` |
| **Агрегация** | Суммирование при группировке по ответственному/источнику; не суммируется по этапам (каждый этап -- отдельное значение) |
| **Единица** | штуки |
| **Формат** | Целое число: `42` |
| **Edge cases** | Если сделка входила на этап несколько раз за период (возврат), считается один раз (DISTINCT). Если сделка перескочила этап -- она НЕ считается вошедшей на пропущенный этап |
| **Бенчмарк** | Зависит от бизнеса; первый этап должен иметь максимальное значение |
| **Пример** | Этап «Квалификация»: 120 сделок вошло за январь |

---

### 1.2. Сумма сделок на этапе

| Поле | Значение |
|------|----------|
| **Название** | Сумма сделок на этапе (Stage deal value) |
| **Определение** | Суммарная стоимость сделок из когорты периода, побывавших на данном этапе |
| **Формула** | `SUM(deal_amount) WHERE card_id IN cohort AND stage = S` |
| **Агрегация** | Суммирование при группировке по измерениям |
| **Единица** | валюта (рубли / доллары / евро -- зависит от настроек компании) |
| **Формат** | Число с разделителями: `1 250 000 руб.` |
| **Edge cases** | Сделки без суммы (`deal_amount IS NULL`) исключаются из SUM, но учитываются в COUNT. Если все сделки без суммы -- отображается `--` (не ноль) |
| **Бенчмарк** | Специфично для компании |
| **Пример** | Этап «Предложение»: 8 500 000 руб. |

---

### 1.3. Средний чек на этапе

| Поле | Значение |
|------|----------|
| **Название** | Средний чек на этапе (Average deal size at stage) |
| **Определение** | Средняя стоимость одной сделки из числа вошедших на этап за период |
| **Формула** | `SUM(deal_amount) / COUNT(card_id WHERE deal_amount IS NOT NULL)` -- делим только на сделки с заполненной суммой |
| **Агрегация** | При группировке -- пересчитывается (взвешенное среднее), не простое среднее средних |
| **Единица** | валюта |
| **Формат** | Число с разделителями: `85 000 руб.` |
| **Edge cases** | Если ни одна сделка не имеет суммы -- `--`. Если одна сделка -- среднее = её сумма. Деление на ноль невозможно при условии наличия хотя бы одной сделки с суммой |
| **Бенчмарк** | B2B SaaS: 50 000 -- 500 000 руб. |
| **Пример** | Этап «Квалификация»: средний чек 127 000 руб. при 40 сделках с суммой из 120 |

---

### 1.4. Конверсия этап -> этап

| Поле | Значение |
|------|----------|
| **Название** | Конверсия этап -> этап (Stage-to-stage conversion rate) |
| **Определение** | Доля сделок из когорты, побывавших на этапе N, которые перешли на этап N+1 (или любой последующий). Показывает «проходимость» каждого этапа |
| **Формула** | `COUNT(DISTINCT card_id WHERE cohort AND entered stage N AND entered any stage > N) / COUNT(DISTINCT card_id WHERE cohort AND entered stage N)` |
| **Уточнение** | Числитель: когортные сделки, которые из этапа N перешли вперёд. Знаменатель: все когортные сделки, побывавшие на этапе N. Сделки, ещё находящиеся на этапе N, включаются в знаменатель, но НЕ в числитель (консервативный подход) |
| **Агрегация** | Не суммируется -- пересчитывается для каждого среза |
| **Единица** | процент |
| **Формат** | `67.5%` (один десятичный знак) |
| **Edge cases** | Если на этап N не вошла ни одна сделка -- `--` (не 0%). Для последнего этапа перед win/loss -- конверсия считается как доля ушедших в win. Перескоки: сделка из этапа 1 сразу в этап 3 засчитывается как конверсия этапа 1 -> далее, но НЕ засчитывается как конверсия этапа 2 |
| **Бенчмарк** | Первые этапы: 40-60%, финальные: 20-40% |
| **Пример** | Квалификация -> Встреча: 80 из 120 = 66.7% |

---

### 1.5. Конверсия этап -> победа (Win rate от этапа)

| Поле | Значение |
|------|----------|
| **Название** | Win rate от этапа (Stage-to-win conversion) |
| **Определение** | Доля сделок из когорты, побывавших на данном этапе, которые в итоге перешли в «Выигран» |
| **Формула** | `COUNT(DISTINCT card_id WHERE cohort AND entered stage S AND entered win_column) / COUNT(DISTINCT card_id WHERE cohort AND entered stage S)` |
| **Уточнение** | Победа учитывается независимо от даты — важно, что сделка из когорты в итоге выиграла. Это позволяет корректно оценивать win rate для когорт, сделки которых ещё не все завершились |
| **Агрегация** | Не суммируется -- пересчитывается |
| **Единица** | процент |
| **Формат** | `23.1%` |
| **Edge cases** | Сделки, ещё находящиеся в воронке (не won и не lost) -- в знаменателе, но не в числителе. Для первого этапа win rate = общая конверсия воронки |
| **Бенчмарк** | B2B: 15-25% от первого этапа, 50-70% от последнего этапа перед win |
| **Пример** | Этап «Предложение»: 30 из 80 перешли в «Выигран» = 37.5% |

---

### 1.6. Среднее время на этапе

| Поле | Значение |
|------|----------|
| **Название** | Среднее время на этапе (Average time in stage) |
| **Определение** | Среднее арифметическое времени нахождения когортных сделок на данном этапе |
| **Формула** | `AVG(exited_at - entered_at) WHERE card_id IN cohort AND stage = S AND exited_at IS NOT NULL` |
| **Уточнение** | Время считается только для завершённых пребываний (сделка покинула этап). Текущие сделки (ещё на этапе) не включаются в среднее, но отображаются отдельно как «В процессе» |
| **Агрегация** | При группировке -- пересчитывается (средневзвешенное) |
| **Единица** | дни (с точностью до десятых) |
| **Формат** | `4.2 дн.` |
| **Edge cases** | Если сделка входила и выходила с этапа несколько раз -- каждое пребывание считается отдельно. Время считается в календарных днях (24 часа = 1 день). Если ни одна сделка не покинула этап -- `--` |
| **Бенчмарк** | Зависит от этапа: первые этапы 1-3 дня, средние 3-7 дней, финальные 5-14 дней |
| **Пример** | Этап «Переговоры»: среднее 6.3 дня на основании 45 сделок |

---

### 1.7. Медианное время на этапе

| Поле | Значение |
|------|----------|
| **Название** | Медианное время на этапе (Median time in stage) |
| **Определение** | Медиана времени нахождения сделки на этапе. Менее чувствительна к выбросам, чем среднее |
| **Формула** | `PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY exited_at - entered_at) WHERE card_id IN cohort AND stage = S AND exited_at IS NOT NULL` |
| **Единица** | дни |
| **Формат** | `3.0 дн.` |
| **Edge cases** | Аналогично среднему. При чётном числе сделок -- среднее двух центральных значений. При одной сделке -- медиана = её время |
| **Бенчмарк** | Обычно медиана на 20-30% ниже среднего (распределение с правым хвостом) |
| **Пример** | Этап «Переговоры»: медиана 4.5 дня (среднее 6.3 дня -- есть выбросы) |

---

### 1.8. Количество зависших сделок

| Поле | Значение |
|------|----------|
| **Название** | Зависшие сделки (Stale deals) |
| **Определение** | Количество сделок, находящихся на данном этапе дольше установленного порога на момент `date_to` |
| **Формула** | `COUNT(DISTINCT card_id) WHERE stage = S AND still_on_stage = true AND (date_to - entered_at) > threshold_days` |
| **Порог** | Настраиваемый параметр `stale_threshold_days` для каждого этапа. Значение по умолчанию: `2 * median_time_in_stage` (если данных для медианы нет -- 14 дней) |
| **Агрегация** | Суммирование при группировке |
| **Единица** | штуки |
| **Формат** | Целое число, выделенное цветом: `7` (красный если > 20% от сделок на этапе, оранжевый если 10-20%, серый если < 10%) |
| **Edge cases** | Если порог не задан и нет данных для автоматического расчёта -- используется 14 дней. Сделки без даты входа на этап -- исключаются |
| **Бенчмарк** | Доля зависших < 10% от текущих сделок на этапе -- норма |
| **Пример** | Этап «Предложение»: 5 сделок зависли (порог 10 дней, максимальное время 34 дня) |

## 2. Метрики всей воронки

Агрегированные показатели, характеризующие воронку целиком.

### 2.1. Общая конверсия воронки

| Поле | Значение |
|------|----------|
| **Название** | Общая конверсия (Overall conversion rate) |
| **Определение** | Доля сделок из когорты периода, которые достигли победы |
| **Формула** | `COUNT(DISTINCT card_id WHERE cohort AND entered win_column) / COUNT(DISTINCT card_id WHERE cohort)` |
| **Период** | Когорта определяется по дате первого входа в воронку. Победа учитывается независимо от даты |
| **Единица** | процент |
| **Формат** | `18.5%` |
| **Edge cases** | Конверсия не может превысить 100% (знаменатель = вся когорта). Для «молодых» когорт конверсия будет ниже реальной (часть сделок ещё в процессе). Сделки, созданные на промежуточном этапе, корректно входят в когорту (см. пример 3.5 в funnel-examples.md) |
| **Бенчмарк** | B2B: 15-25%, B2C: 2-5% |
| **Пример** | 200 сделок вошло, 37 выиграно = 18.5% |

---

### 2.2. Средний цикл сделки

| Поле | Значение |
|------|----------|
| **Название** | Средний цикл сделки (Average sales cycle) |
| **Определение** | Среднее время от первого входа в воронку до перехода в «Выигран» |
| **Формула** | `AVG(win_date - first_entered_at) WHERE card_id IN cohort AND outcome = 'won'` |
| **Уточнение** | Считается только для выигранных сделок когорты. `first_entered_at` = дата-якорь (первый вход на любой этап). Победа учитывается независимо от даты |
| **Единица** | дни |
| **Формат** | `34.2 дн.` |
| **Edge cases** | Если нет выигранных сделок за период -- `--`. Сделки, выигранные в тот же день, когда вошли -- цикл = 0 дней (корректно для быстрых сделок) |
| **Бенчмарк** | B2B SaaS: 30-90 дней, B2B Enterprise: 90-180 дней |
| **Пример** | 37 выигранных сделок, средний цикл 42.7 дней |

---

### 2.3. Объём воронки (Pipeline value)

| Поле | Значение |
|------|----------|
| **Название** | Объём воронки (Pipeline value) |
| **Определение** | Суммарная стоимость всех активных сделок, находящихся на этапах воронки на момент `date_to` |
| **Формула** | `SUM(deal_amount) WHERE stage IN stage_order AND still_on_stage = true AS OF date_to` |
| **Уточнение** | «Активные» = не перешли в win или loss. Это snapshot-метрика: показывает состояние на конец периода |
| **Единица** | валюта |
| **Формат** | `12 450 000 руб.` |
| **Edge cases** | Сделки без суммы -- исключаются из SUM. Рядом отображается количество сделок без суммы: `12 450 000 руб. (3 сделки без суммы)` |
| **Бенчмарк** | Pipeline должен быть в 3-5 раз больше квартального плана продаж |
| **Пример** | 95 активных сделок на общую сумму 12 450 000 руб. |

---

### 2.4. Взвешенный объём воронки

| Поле | Значение |
|------|----------|
| **Название** | Взвешенный объём (Weighted pipeline value) |
| **Определение** | Сумма стоимостей активных сделок, взвешенных по вероятности победы на текущем этапе |
| **Формула** | `SUM(deal_amount * stage_win_probability) WHERE stage IN stage_order AND still_on_stage = true` |
| **Уточнение** | `stage_win_probability` -- историческая конверсия этап -> win, рассчитанная на основе прошлых данных (или заданная вручную в настройках воронки). Если исторических данных нет -- используется линейная интерполяция: вероятность = `stage_index / total_stages` |
| **Единица** | валюта |
| **Формат** | `4 230 000 руб.` |
| **Edge cases** | Если вероятности не заданы и нет исторических данных -- отображается предупреждение. Сделки без суммы -- исключаются |
| **Бенчмарк** | Обычно 25-40% от pipeline value |
| **Пример** | Pipeline 12.45 млн, взвешенный 4.23 млн (34%) |

**Таблица вероятностей по умолчанию (линейная интерполяция для 5 этапов):**

| Этап | Позиция | Вероятность |
|------|---------|-------------|
| Квалификация | 1/5 | 20% |
| Встреча | 2/5 | 40% |
| Предложение | 3/5 | 60% |
| Переговоры | 4/5 | 80% |
| (Win) | 5/5 | 100% |

---

### 2.5. Скорость воронки (Pipeline velocity)

| Поле | Значение |
|------|----------|
| **Название** | Скорость воронки (Pipeline velocity) |
| **Определение** | Скорость генерации выручки воронкой. Показывает, сколько денег «производит» воронка в единицу времени |
| **Формула** | `(deals_count * win_rate * avg_deal_size) / avg_sales_cycle` |
| **Переменные** | `deals_count` = размер когорты периода; `win_rate` = общая конверсия когорты (2.1); `avg_deal_size` = средний чек выигранных сделок когорты; `avg_sales_cycle` = средний цикл когорты (2.2) в днях |
| **Единица** | валюта / день |
| **Формат** | `142 500 руб./день` |
| **Edge cases** | Если `avg_sales_cycle = 0` -- невозможно рассчитать, отображается `--`. Если нет выигранных сделок -- `win_rate = 0`, velocity = 0 |
| **Бенчмарк** | Специфично для компании; важен тренд (рост velocity = улучшение продаж) |
| **Пример** | 200 сделок * 18.5% * 230 000 руб. / 42.7 дней = 199 297 руб./день |

---

### 2.6. Количество потерянных сделок

| Поле | Значение |
|------|----------|
| **Название** | Потерянные сделки (Lost deals) |
| **Определение** | Количество сделок из когорты периода, перешедших в «Проигран» |
| **Формула** | `COUNT(DISTINCT card_id) WHERE card_id IN cohort AND outcome = 'lost'` |
| **Агрегация** | Суммирование при группировке. Дополнительно -- разбивка по последнему активному этапу перед проигрышем |
| **Единица** | штуки |
| **Формат** | `63` |
| **Edge cases** | Сделка, перешедшая из loss обратно в воронку и снова в loss -- считается один раз (по последнему переходу). Сделки, ушедшие в loss с этапа, не входящего в `stage_order` -- учитываются с пометкой «этап не определён» |
| **Бенчмарк** | Зависит от бизнеса; важно соотношение lost / (won + lost) |
| **Пример** | 63 сделки проиграны, из них 28 с этапа «Квалификация», 20 с «Предложения», 15 с «Переговоров» |

---

### 2.7. Коэффициент потерь по этапу (Drop-off rate)

| Поле | Значение |
|------|----------|
| **Название** | Коэффициент потерь по этапу (Stage drop-off rate) |
| **Определение** | Доля когортных сделок, побывавших на этапе S, которые были потеряны (ушли в loss) именно с этого этапа |
| **Формула** | `COUNT(card_id WHERE cohort AND last_active_stage = S AND outcome = 'lost') / COUNT(card_id WHERE cohort AND entered stage S)` |
| **Уточнение** | `last_active_stage` -- последний этап из `stage_order`, на котором находилась сделка перед переходом в loss |
| **Единица** | процент |
| **Формат** | `15.2%` |
| **Edge cases** | Сумма drop-off rate по всем этапам + общая конверсия + доля «ещё в процессе» = 100%. Если с этапа никто не уходил в loss -- 0% |
| **Бенчмарк** | Высокий drop-off на ранних этапах (30-40%) -- нормально. Высокий drop-off на финальных этапах (> 20%) -- проблема |
| **Пример** | Этап «Предложение»: 20 из 80 ушли в loss = 25% drop-off |

## 3. Метрики сравнения периодов

Рассчитываются для каждой метрики из разделов 1 и 2 при выборе двух периодов.

### 3.1. Абсолютная дельта

| Поле | Значение |
|------|----------|
| **Название** | Абсолютная дельта (Absolute delta) |
| **Определение** | Разница между значением метрики в текущем периоде и предыдущем |
| **Формула** | `metric_current - metric_previous` |
| **Единица** | Та же, что у исходной метрики |
| **Формат** | Со знаком: `+12`, `-3.5%`, `+250 000 руб.` Зелёный цвет для положительных изменений (рост конверсии, снижение цикла), красный -- для негативных |
| **Edge cases** | Если метрика в одном из периодов = `--` (нет данных) -- дельта = `--`. Для метрик «время» и «зависшие» положительная дельта = ухудшение (выделяется красным) |
| **Пример** | Конверсия: 18.5% - 15.2% = +3.3 п.п. |

---

### 3.2. Относительная дельта

| Поле | Значение |
|------|----------|
| **Название** | Относительная дельта (Relative delta, %) |
| **Определение** | Процентное изменение метрики относительно предыдущего периода |
| **Формула** | `(metric_current - metric_previous) / metric_previous * 100%` |
| **Единица** | процент |
| **Формат** | `+21.7%` |
| **Edge cases** | Если `metric_previous = 0` -- дельта = `--` (деление на ноль). Если `metric_previous = --` -- дельта = `--`. Логика цвета аналогична абсолютной дельте |
| **Пример** | Сделок на этапе: 120 vs. 98 = +22.4% |

---

### 3.3. Тренд

| Поле | Значение |
|------|----------|
| **Название** | Тренд (Trend indicator) |
| **Определение** | Направление изменения метрики: растёт, падает или стабильно |
| **Формула** | Если `abs(relative_delta) < 5%` -- «стабильно». Если `relative_delta >= 5%` -- «растёт». Если `relative_delta <= -5%` -- «падает» |
| **Порог** | 5% -- значение по умолчанию, настраиваемое |
| **Единица** | enum: `up` / `down` / `stable` |
| **Формат** | Иконка со стрелкой: вверх (зелёная), вниз (красная), вправо (серая). Для метрик времени/зависших -- инвертированная логика цвета |
| **Edge cases** | Если одна из метрик = `--` -- тренд = `--` |
| **Пример** | Конверсия +21.7% -> тренд «растёт» (стрелка вверх, зелёная) |

## 5. Измерения (Dimensions) для группировки

Все метрики могут быть разбиты по следующим измерениям:

| Измерение | Источник | Описание |
|-----------|----------|----------|
| Этап воронки | `stage_order` | Каждый этап -- отдельная строка/столбец |
| Ответственный | `card.responsible_id` | Менеджер по продажам |
| Источник | `card.custom_field[source]` | Откуда пришла сделка |
| Период | `date_from..date_to` | Неделя, месяц, квартал |
| Сумма (бакеты) | `card.custom_field[amount]` | Малые (< 100k), средние (100k-500k), крупные (> 500k) |
| Теги | `card.tags` | Произвольные теги |
