# Примеры работы воронки продаж

## Общее описание

Документ содержит наглядные примеры работы отчёта «Воронка продаж»: как сделки двигаются по этапам, как это отражается на графике и метриках, и как период фильтрации влияет на отображение.

Все примеры используют одну воронку:

```
Квалификация → Встреча → Предложение → Переговоры → Выигран / Проигран
```

---

## 1. Примеры движения сделок

### 1.1. Нормальная сделка — прошла всю воронку

```
Сделка «ООО Рога» (500 000 руб.)

1 июня  → попала в "Квалификацию"
5 июня  → менеджер перетащил в "Встречу"
12 июня → перетащил в "Предложение"
20 июня → перетащил в "Переговоры"
28 июня → перетащил в "Выигран" ✅
```

| Метрика | Влияние |
|---------|---------|
| Квалификация: кол-во | +1 |
| Встреча: кол-во | +1 |
| Предложение: кол-во | +1 |
| Переговоры: кол-во | +1 |
| Общая конверсия | +1 в числитель и знаменатель |
| Цикл сделки | 27 дней (28 июня - 1 июня) |
| Время на «Квалификации» | 4 дня |
| Время на «Встрече» | 7 дней |
| Время на «Предложении» | 8 дней |
| Время на «Переговорах» | 8 дней |

### 1.2. Сделка отвалилась на середине

```
Сделка «ИП Иванов» (100 000 руб.)

3 июня  → попала в "Квалификацию"
8 июня  → менеджер перетащил в "Встречу"
15 июня → перетащил в "Проигран" ❌
```

| Метрика | Влияние |
|---------|---------|
| Квалификация: кол-во | +1 |
| Встреча: кол-во | +1 |
| Предложение: кол-во | не считается (сделка туда не входила) |
| Конверсия «Квалиф → Встреча» | +1 в числитель и знаменатель |
| Конверсия «Встреча → Предложение» | +1 только в знаменатель (не прошла дальше) |
| Drop-off «Встреча» | +1 (потеряна с этого этапа) |
| Потерянные сделки | +1, последний этап = «Встреча» |

### 1.3. Сделка перескочила этап

```
Сделка «Газпром» (5 000 000 руб.)

2 июня  → попала в "Квалификацию"
4 июня  → менеджер перетащил сразу в "Предложение" (пропустив "Встречу")
10 июня → "Переговоры"
18 июня → "Выигран" ✅
```

| Метрика | Влияние |
|---------|---------|
| Квалификация: кол-во | +1 |
| Встреча: кол-во | **не считается** (сделка туда не входила) |
| Предложение: кол-во | +1 |
| Переговоры: кол-во | +1 |
| Конверсия «Квалиф → далее» | +1 в числитель (сделка прошла дальше) |
| Конверсия «Встреча → далее» | **не затронута** (сделка не входила на «Встречу») |

**Правило:** запись в `funnel_deal_history` создаётся только для этапов, на которые сделка реально попадала. Пропущенные этапы игнорируются.

### 1.4. Сделка вернулась назад

```
Сделка «Сбербанк» (2 000 000 руб.)

5 июня  → "Квалификация"
10 июня → "Встреча"
15 июня → "Предложение"
18 июня → вернули обратно в "Встречу" (клиент попросил другие условия)
22 июня → снова в "Предложение"
25 июня → "Переговоры"
```

| Метрика | Влияние |
|---------|---------|
| Встреча: кол-во сделок | **1** (DISTINCT card_id — не 2) |
| Встреча: кол-во визитов | 2 (visit_number = 1 и 2) |
| Встреча: время | Два наблюдения: 5 дней (10–15 июня) и 4 дня (18–22 июня) |
| Встреча: сумма | 2 000 000 руб. — берётся из первого визита, один раз |
| Конверсия «Встреча → далее» | +1 в числитель (хотя бы раз прошла дальше) |

**Правило:** для количества и суммы — DISTINCT. Для времени — каждый визит считается отдельным наблюдением.

### 1.5. Сделка двигается назад-вперёд многократно

```
Сделка «ТехноПром» (700 000 руб.)

1 июня  → "Квалификация"
3 июня  → "Встреча"
5 июня  → "Предложение"
8 июня  → назад в "Встречу"     (доп. встреча с клиентом)
11 июня → "Предложение"
14 июня → назад во "Встречу"    (ещё одна встреча)
17 июня → "Предложение"
20 июня → "Переговоры"
```

| Этап | Кол-во сделок | Визитов | Время (наблюдения) |
|------|:---:|:---:|---|
| Квалификация | 1 | 1 | 2 дня |
| Встреча | 1 | 3 | 2 дн. + 3 дн. + 3 дн. |
| Предложение | 1 | 3 | 3 дн. + 3 дн. + 3 дн. |
| Переговоры | 1 | 1 | ещё на этапе |

Конверсия «Встреча → далее» = 1 из 1 = 100% (сделка в итоге прошла дальше).

### 1.6. Быстрая сделка — выиграли за один день

```
Сделка «ООО Ракета» (50 000 руб.)

10 июня → попала в "Квалификацию"
10 июня → менеджер перетащил в "Встречу"
10 июня → перетащил в "Предложение"
10 июня → перетащил в "Выигран" ✅
```

| Метрика | Влияние |
|---------|---------|
| Квалификация: кол-во | +1 |
| Встреча: кол-во | +1 |
| Предложение: кол-во | +1 |
| Переговоры: кол-во | **0** (пропущены) |
| Цикл сделки | 0 дней |
| Время на каждом этапе | ≈ 0 (тянет среднее вниз) |

**Правило:** цикл сделки = 0 дней — это корректно для быстрых продаж. Порядок перемещений внутри одного дня определяется по `id` записи (tiebreaker).

### 1.7. Зависшая сделка — стоит на этапе неделями

```
Сделка «Минстрой» (8 000 000 руб.)

2 июня → попала в "Предложение"
... тишина ...
30 июня → всё ещё на "Предложении" (28 дней без движения)
```

| Метрика | Влияние |
|---------|---------|
| Предложение: кол-во | +1 |
| Предложение: конверсия | +1 в знаменатель, **не** в числитель (ещё не прошла) |
| Предложение: время | **не включается** в среднее (сделка не покинула этап) |
| Предложение: зависшие | +1 (28 дней > порог 10 дней) |
| Pipeline value | +8 000 000 руб. (сделка активна) |

**Правило:** зависшие — это snapshot на `date_to`. Порог по умолчанию = `2 × медианное время на этапе` или 14 дней, если данных нет.

### 1.8. Проигранная сделка вернулась в воронку

```
Сделка «Альфа-Банк» (1 200 000 руб.)

3 июня  → "Квалификация"
7 июня  → "Встреча"
12 июня → "Проигран" ❌ (клиент отказался)
20 июня → менеджер вернул в "Встречу" (клиент передумал)
25 июня → "Предложение"
```

| Период | Outcome сделки |
|--------|----------------|
| На 12 июня | `lost` |
| На 20 июня | `in_progress` (вернулась в воронку) |
| На 30 июня | `in_progress` (на «Предложении») |

| Метрика | Влияние |
|---------|---------|
| Встреча: визитов | 2 (первый: 7–12 июня, второй: 20–25 июня) |
| Встреча: кол-во сделок | 1 (DISTINCT) |
| Потерянные сделки | **не считается** — итоговый outcome не `lost` |

**Правило:** outcome определяется по последнему состоянию. Сделка, вернувшаяся из «Проигран», больше не считается потерянной.

---

### 1.9. Сделка ушла в колонку вне воронки

```
Сделка «МегаФон» (900 000 руб.)

5 июня  → "Квалификация"
10 июня → "Встреча"
15 июня → менеджер перетащил в "На паузе" (колонка вне воронки)
```

| Метрика | Влияние |
|---------|---------|
| Встреча: время | 5 дней (10–15 июня), пребывание закрыто |
| Конверсия «Встреча → далее» | +1 в знаменатель, **не** в числитель |
| Outcome | `in_progress` (не lost, но покинула воронку) |
| Pipeline value | **не включается** (сделка не на этапе воронки) |

**Правило:** перемещение в колонку, не входящую в `stage_order`, `win_column_ids` или `loss_column_ids`, закрывает пребывание на этапе. Сделка «покидает воронку», но не считается проигранной.

Если менеджер позже вернёт карточку в «Предложение» — это будет новый вход на этап.

### 1.10. Пачка сделок с рекламной кампании

```
15 июня — запустили рекламу. За 5 дней (15–20 июня):

40 новых карточек создано в "Квалификации"

К 30 июня из этих 40:
  25 → перешли во "Встречу"
  10 → всё ещё на "Квалификации"
  5  → ушли в "Проигран" (нецелевые)
```

| Метрика (только по этим 40 сделкам) | Значение |
|------|---------|
| Квалификация: кол-во | +40 |
| Встреча: кол-во | +25 |
| Конверсия «Квалиф → Встреча» | 25 / 40 = 62.5% |
| Drop-off «Квалификация» | 5 / 40 = 12.5% |
| Ещё на «Квалификации» | 10 сделок (в знаменателе конверсии, но не в числителе) |

При сравнении с маем (без рекламы) — дельта по количеству на «Квалификации» покажет рост.

### 1.11. Сделка без суммы

```
Сделка «Стартап X» (сумма не указана)

8 июня  → "Квалификация"
12 июня → "Встреча"
18 июня → "Предложение"
```

| Метрика | Влияние |
|---------|---------|
| Кол-во на каждом этапе | +1 (сумма не влияет) |
| Сумма на этапе | **не включается** в SUM |
| Средний чек | **не включается** (делим только на сделки с суммой) |
| Pipeline value | не включается, пометка: `(1 сделка без суммы)` |
| Взвешенный pipeline | не включается |
| Конверсия | +1 нормально (считается по штукам) |
| Velocity | не влияет на avg_deal_size |

**Правило:** сделки без суммы (`deal_amount IS NULL`) учитываются в количественных метриках, но исключаются из денежных. Если **все** сделки без суммы — денежные метрики отображают `--` (не ноль).

### 1.12. Две сделки одного клиента

```
Сделка «Лукойл — контракт А» (3 000 000 руб.)
  5 июня → "Квалификация" → 10 июня → "Встреча" → 20 июня → "Выигран" ✅

Сделка «Лукойл — контракт Б» (1 500 000 руб.)
  5 июня → "Квалификация" → 8 июня → "Проигран" ❌
```

Это **две разные карточки** — две разные сделки. Воронка работает на уровне карточек, не клиентов.

| Метрика | Значение |
|---------|---------|
| Квалификация: кол-во | 2 |
| Квалификация: сумма | 4 500 000 руб. |
| Встреча: кол-во | 1 (только контракт А) |
| Общая конверсия (по этим двум) | 1 из 2 = 50% |

### 1.13. Сделка создана до периода, завершена в периоде

```
Сделка «РЖД» (10 000 000 руб.)

20 мая  → попала в "Квалификацию"   ← первый вход в воронку = 20 мая
25 мая  → "Встреча"
3 июня  → "Предложение"
15 июня → "Переговоры"
28 июня → "Выигран" ✅
```

Период отчёта — **июнь (1–30 июня)**.

**Когорта:** первый вход в воронку = 20 мая → сделка принадлежит **когорте мая**, не июня.

| Период (когорта) | Сделка видна? | Какие этапы показываются? |
|---|:---:|---|
| Май | Да | Все: Квалификация, Встреча, Предложение, Переговоры, Выигран |
| Июнь | **Нет** | Не входит в когорту июня |

В **майском** отчёте сделка покажет **полный** путь — все 4 этапа плюс победу. Даже несмотря на то, что «Предложение», «Переговоры» и «Выигран» произошли в июне.

В **июньском** отчёте этой сделки **не будет** — она не входит в июньскую когорту.

**Правило:** принадлежность к периоду определяется по дате-якорю = `MIN(entered_at)`. Дата создания карточки не имеет значения. Для когортных сделок показываются все этапы, которые они прошли.

### 1.14. Сделка на этапе на момент конца периода (snapshot)

```
30 июня, конец дня. На доске находятся:

"Квалификация":  15 сделок, сумма 2 100 000 руб.
"Встреча":       8 сделок, сумма 3 400 000 руб.
"Предложение":   12 сделок, сумма 7 800 000 руб.
"Переговоры":    5 сделок, сумма 4 200 000 руб.
```

**Pipeline value** = 2.1 + 3.4 + 7.8 + 4.2 = **17 500 000 руб.**

**Взвешенный pipeline** (с вероятностями по позиции этапа):

| Этап | Сумма | Позиция | Вероятность | Взвешенная сумма |
|------|------:|:---:|:---:|------:|
| Квалификация | 2 100 000 | 1/4 | 25% | 525 000 |
| Встреча | 3 400 000 | 2/4 | 50% | 1 700 000 |
| Предложение | 7 800 000 | 3/4 | 75% | 5 850 000 |
| Переговоры | 4 200 000 | 4/4 | 100% | 4 200 000 |
| **Итого** | **17 500 000** | | | **12 275 000** |

**Правило:** pipeline и взвешенный pipeline — snapshot-метрики. Они показывают состояние на `date_to`, а не поток за период.

## 2. Влияние периода на отображение воронки

### 2.1. Ключевое правило: когортный подход

Отчёт использует **когортный подход** к фильтрации по периоду.

**Дата-якорь** каждой сделки = дата первого входа на любой этап воронки (`MIN(entered_at)`).

**Когорта периода** = все сделки, чья дата-якорь попадает в `[date_from, date_to]`.

Для сделок из когорты показываются **все** этапы, которые они прошли — независимо от дат перемещений.

Это значит:
- Каждая сделка принадлежит **ровно одному** периоду (тому, где был первый вход)
- Числа на этапах **убывают** сверху вниз (каждый следующий этап ≤ предыдущего)
- По любому числу можно провалиться и увидеть **конкретные карточки**
- Данные **сходятся** с тем, что видно на доске

**Snapshot-метрики** (pipeline, зависшие) — по-прежнему показывают состояние на `date_to`, независимо от когорты.

| Тип метрики | Как фильтруется | Зависит от ширины периода | Примеры |
|-------------|-----------------|:---:|---------|
| Когортная | По дате первого входа в воронку | Да | Кол-во, конверсия, время, drop-off, потери, velocity |
| Snapshot | По состоянию на `date_to` | Нет | Pipeline value, взвешенный pipeline, зависшие |

### 2.2. Одна доска — разные периоды

Дата отчёта: 30 июня.

```
Когорта июня (120 сделок впервые вошли в воронку в июне):
  120 побывали на "Квалификации"
  80 дошли до "Встречи"
  60 дошли до "Предложения"
  30 дошли до "Переговоров"
  20 выиграно, 63 проиграно, 37 ещё в процессе

Когорта последней недели (18 сделок впервые вошли 24–30 июня):
  18 побывали на "Квалификации"
  10 дошли до "Встречи"
  4 дошли до "Предложения"
  1 дошла до "Переговоров"
  0 выиграно, 3 проиграно, 15 ещё в процессе

Когорта сегодня (3 сделки впервые вошли 30 июня):
  3 на "Квалификации" (Яндекс, VK, МТС)
  0 дальше (слишком свежие)
```

#### Период: «Июнь» (1–30 июня)

```
Квалификация   ████████████████████████  120 сделок    8 500 000 руб.
                      ↓ 66.7%
Встреча        ████████████████          80 сделок     6 200 000 руб.
                      ↓ 75%
Предложение    ████████████              60 сделок     4 800 000 руб.
                      ↓ 50%
Переговоры     ██████                    30 сделок     2 900 000 руб.

Когорта: 120 сделок  |  Конверсия: 16.7%  |  Цикл: 42.7 дн.  |  Pipeline: 17.5 млн
```

Полная когорта. Числа убывают сверху вниз — 120 → 80 → 60 → 30. Каждое число — подмножество предыдущего. Можно провалиться и увидеть конкретные 80 карточек на «Встрече».

#### Период: «Эта неделя» (24–30 июня)

```
Квалификация   ████████                  18 сделок     1 200 000 руб.
                      ↓ 55.6%
Встреча        █████                     10 сделок       780 000 руб.
                      ↓ 40%
Предложение    ██                        4 сделки        320 000 руб.
                      ↓ 25%
Переговоры     █                         1 сделка        150 000 руб.

Когорта: 18 сделок  |  Конверсия: 0%  |  Цикл: --  |  Pipeline: 17.5 млн
```

Маленькая когорта. Конверсия 0% — не потому что плохо, а потому что сделки слишком свежие (вошли неделю назад, ещё не успели дойти до победы). Pipeline тот же — это snapshot.

#### Период: «Сегодня» (30 июня)

```
Квалификация   ██                        3 сделки        1 900 000 руб.
                      ↓ 0%
Встреча                                  0 сделок                --
Предложение                              0 сделок                --
Переговоры                               0 сделок                --

Когорта: 3 сделки  |  Конверсия: --  |  Цикл: --  |  Pipeline: 17.5 млн
```

3 сделки только что вошли в воронку — ещё никуда не продвинулись. Это ожидаемо: когорта «сегодня» полезна для подсчёта входящих, а не для анализа конверсий.

| Средний цикл | `--` | `--` | 42.7 дн. | Нет побед в свежих когортах |
| Velocity | `--` | `--` | 199 000/дн. | Нет данных для расчёта |

**Ключевое отличие от событийного подхода:** конверсия для свежих когорт = 0% или `--`, а не завышенное значение. Это корректно: сделки, вошедшие неделю назад, ещё не успели дойти до конца воронки. По мере завершения сделок конверсия когорты растёт.

---

### 2.4. Пример: сделка принадлежит ровно одному периоду

```
Сделка «РЖД» (10 000 000 руб.)

25 мая  → "Встреча"   ← первый вход = 25 мая → когорта мая
3 июня  → "Предложение"
28 июня → "Выигран" ✅
```

| Метрика | Отчёт за май (когорта мая) | Отчёт за июнь (когорта июня) |
|---------|:---:|:---:|
| Сделка входит в когорту? | **Да** (первый вход 25 мая) | **Нет** |
| Встреча: кол-во | +1 | — |
| Предложение: кол-во | +1 (хотя вход 3 июня) | — |
| Победы | +1 (хотя победа 28 июня) | — |
| Pipeline (на конец периода) | включена (на «Встрече» 31 мая) | не включена (уже в «Выигран») |

**Ключевое отличие:** сделка принадлежит **только** майской когорте. В майском отчёте видны **все** её этапы — включая те, что произошли в июне. В июньском отчёте этой сделки нет.

---

### 2.5. Пример: сравнение двух периодов

Пользователь выбрал: **июнь vs. май**.

```
                          Июнь       Май       Абс. дельта    Отн. дельта    Тренд

Квалификация              120        98        +22            +22.4%         ↑ зелёный
Встреча                    80        75        +5             +6.7%          ↑ зелёный
Предложение                60        70        -10            -14.3%         ↓ красный
Переговоры                 30        32        -2             -6.3%          ↓ красный
Выигран                    20        15        +5             +33.3%         ↑ зелёный

Общая конверсия          18.5%     15.2%       +3.3 п.п.     +21.7%         ↑ зелёный
Средний цикл             42.7 дн.  38.1 дн.   +4.6 дн.      +12.1%         ↓ красный
Pipeline                 17.5 млн  14.2 млн    +3.3 млн      +23.2%         ↑ зелёный
Velocity                 199 000   132 000     +67 000        +50.8%         ↑ зелёный
Потерянные                 63        55        +8             +14.5%         ↓ красный
```

**Что видит руководитель:**
- Больше сделок входит (хорошо)
- Конверсия выросла (хорошо)
- Но на «Предложении» провал — -14.3% (нужно разобраться)
- Цикл удлинился — сделки стали закрываться медленнее (плохо)
- Потерь больше — абсолютно логично, если больше входящих (нужно смотреть в процентах)

Автоматический расчёт предыдущего периода: `previous_date_from = date_from - (date_to - date_from + 1)`.

---

### 2.6. Рекомендации по выбору периода

| Период | Для чего подходит | Надёжность статистики |
|--------|-------------------|:---:|
| Сегодня | Оперативный контроль: «что произошло сегодня?» | Низкая |
| Эта неделя | Тактика: «хватает ли активности?» | Средняя |
| Месяц | Аналитика: «где узкие места воронки?» | Высокая |
| Квартал | Стратегия: «какой тренд? растём или нет?» | Высокая |
| Произвольный | Анализ конкретной кампании, события, эксперимента | Зависит от объёма |

Чем **уже** период — тем свежее данные, но тем больше случайных колебаний. Чем **шире** — тем надёжнее выводы, но тем больше «размыта» картина.

## 3. Особые случаи (edge cases)

### 3.1. Пустой период

Выбран период, в котором не было ни одного перемещения.

```
Квалификация   0 сделок    --
Встреча        0 сделок    --
Предложение    0 сделок    --
Переговоры     0 сделок    --

Сообщение: «Нет данных за выбранный период»
Pipeline: 17.5 млн (snapshot всё равно показывается)
```

---

### 3.2. Воронка с одним этапом

Если в настройках воронки указан только один этап:

```
Переговоры     30 сделок    2 900 000 руб.
```

- Конверсия этап → этап: `--` (нет следующего этапа)
- Конверсия в win: рассчитывается нормально
- Все остальные метрики работают

---

### 3.3. Одновременные перемещения (batch move)

Менеджер выделил 10 карточек и перетащил их разом из «Квалификации» во «Встречу». Все 10 записей получат одинаковый `moved_at`.

**Правило:** порядок определяется по `id` записи как tiebreaker. Каждое перемещение обрабатывается отдельно.

---

### 3.4. Удалённая / архивная карточка

| Состояние | Учитывается? |
|-----------|:---:|
| Активная карточка | Да |
| Архивная карточка (`include_archived = false`, по умолчанию) | Нет |
| Архивная карточка (`include_archived = true`) | Да |
| Удалённая карточка (если `card_movements` сохранены) | Да |
| Удалённая карточка (если `card_movements` удалены каскадно) | Нет |

---

### 3.5. Сделка создана сразу на промежуточном этапе

```
Сделка «Минстрой» (8 000 000 руб.)

2 июня  → карточка создана сразу в "Предложении" (3-й этап)
10 июня → "Переговоры"
18 июня → "Выигран" ✅
```

Сделка никогда не была ни на «Квалификации», ни на «Встрече».

| Этап | Кол-во | Конверсия в след. этап | Сумма |
|------|:---:|:---:|:---:|
| Квалификация | — | — | — |
| Встреча | — | — | — |
| Предложение | +1 | +1 в числитель и знаменатель | +8 000 000 |
| Переговоры | +1 | +1 в числитель и знаменатель | +8 000 000 |

| Метрика | Влияние |
|---------|---------|
| Квалификация: кол-во | **0** — сделка туда не входила |
| Встреча: кол-во | **0** — сделка туда не входила |
| Конверсия «Квалиф → Встреча» | **не затронута** |
| Конверсия «Встреча → Предложение» | **не затронута** |
| Общая конверсия | +1 в числитель (выиграна) и +1 в знаменатель (вошла хотя бы на один этап) |
| Победы | +1 |
| Цикл сделки | 16 дней (18 июня − 2 июня) |

**Правило:** когорта определяется по дате первого входа на **любой** этап воронки. Сделка, созданная сразу на «Предложении», входит в когорту по дате создания на «Предложении» (= её первый вход). Общая конверсия = выигранные из когорты / размер когорты — не может превысить 100%.

Когда это происходит на практике:
- Горячий лид от руководства — сразу в «Предложение»
- Импорт сделок из другой системы
- Менеджер создаёт карточку на этапе, соответствующем текущему статусу клиента
