# Мастер-план: Отчёт «Воронка продаж» для Kaiten CRM

## Цель

Разработать отчёт «Воронка продаж», который визуализирует прохождение сделок через этапы воронки и помогает руководителям отделов продаж принимать решения на основе данных.

## Фазы реализации

### Фаза 1. Дизайн UI (1-2 недели)

**Цель:** спроектировать интерфейс отчёта на дизайн-системе shadcn.

**Задачи:**
1. Wireframes основных экранов (отчёт, drill-down, панель настроек)
2. Компонент трапецеидальной funnel chart (кастомный, на базе Recharts)
3. Панель фильтров
4. Drill-down боковая панель со списком сделок
5. Режим сравнения периодов (side-by-side)
6. `BestGuessAlert` -- компонент алерта при неоднозначности автоконфигурации (F-002)
7. `MetricModeToggle` -- переключатель "Количество / Сумма" (F-004)
8. Multi-board конфигурация: отображение этапов с нескольких досок, резолвинг конфликтов имён
9. Адаптивность (desktop + tablet)

### Фаза 2. Backend -- модель данных и API (2-3 недели)

**Задачи:**
1. Миграции БД: `funnel_config` (с `space_id` вместо `board_id`, `metric_mode`, `auto_generated`, `board_ids[]`), `funnel_deal_history` (с `board_id`), `funnel_deal_summary` (с `board_id`)
2. Алгоритм Best Guess автоконфигурации (по спецификации из `research/best-guess-algorithm.md`)
3. Multi-board загрузка и линеаризация колонок (по спецификации из `data-model/multi-board-unification.md`)
4. Сервис построения `funnel_deal_history` из `card_movements` (multi-board)
5. Инкрементальное обновление при новых перемещениях
6. API endpoints (по спецификации из `data-model/api-spec.md`), включая `auto-config`
7. Фильтрация, пагинация, сортировка
8. Экспорт CSV/XLSX
9. Rate limiting и кеширование

### Фаза 3. Frontend -- реализация UI (2-3 недели)

**Задачи:**
1. Страница отчёта с маршрутизацией
2. Панель корректировки автоконфигурации (`FunnelConfigPanel` -- боковая панель с включением/исключением досок, этапов, drag-and-drop, выбор won/lost)
3. Переключатель метрики `MetricModeToggle` (count / amount)
4. Funnel chart + bar chart + таблица (переключение)
5. Панель фильтров (период, ответственный, теги, сумма)
6. `BestGuessAlert` -- алерт-баннер над графиком при неоднозначности автоконфигурации
7. Drill-down панель (Sheet) со списком сделок
8. Режим сравнения периодов
9. Экспорт (CSV, XLSX, PNG)
10. Аналитические события (трекинг использования)

### Фаза 4. Тестирование и запуск (1 неделя)

**Задачи:**
1. Unit-тесты расчётов (edge cases из `calculation-spec.md`)
2. Integration-тесты API
3. E2E тесты ключевых сценариев
4. Performance-тестирование (объёмы из `data-model.md`)
5. Ревью accessibility
6. Поэтапный rollout

## Ключевые решения

> Обновлено по результатам ревью CPO (февраль 2026). Пункты 3, 6, 7 -- новые/изменённые.

1. **Трапецеидальная funnel chart** -- основной вид визуализации (конкурентное преимущество)
2. **Сравнение периодов side-by-side с дельтами** -- ни один конкурент не делает хорошо
3. **Best Guess -- график строится сразу, без настройки** -- паттерн Kaiten: все графики работают на Best Guess (как кумулятивная диаграмма). Экран настройки удалён, корректировка опциональна через боковую панель. Источник: P-001, P-002, F-001
4. **Drill-down обязателен** -- клик по этапу -> список сделок -> переход в карточку
5. **4 базовые метрики MVP** -- количество, сумма, конверсия, время на этапе
6. **Единая воронка на пространство (multi-board)** -- воронка собирает колонки со всех досок пространства в единую последовательность. `space_id` вместо `board_id`. Источник: P-005, F-005
7. **Count по умолчанию, amount опционально** -- воронка всегда строится по количеству карточек. Переключатель на сумму доступен, если числовое поле определено. Fallback на count при конфликте полей или их отсутствии. Источник: P-006, F-004

| # | Вопрос | Решение |
|---|--------|---------|
