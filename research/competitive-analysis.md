# Конкурентный анализ: Отчёт «Воронка продаж»

## Цель исследования

Понять, как ведущие CRM и PM-системы реализуют отчёт «Воронка продаж», чтобы определить лучшие практики, типичные паттерны и возможности для дифференциации Kaiten.

---

## 1. Bitrix24

### Общее описание

Bitrix24 предлагает раздел «CRM-аналитика» с двумя типами воронки: **классическая** и **конверсионная**. Отчёт объединяет воронку лидов и сделок в единый интерфейс. Фильтрация доступна по любым полям, включая пользовательские.

### Визуализация

| Характеристика | Описание |
|---|---|
| Тип диаграммы | Вертикальная столбчатая диаграмма (bar chart) |
| Альтернативные виды | Таблица с цифрами, карточки-дашборды с ключевыми показателями |
| Интерактивность | При наведении на столбец — всплывающее окно с конверсией, суммой, средней длительностью этапа |
| Цветовая схема | Цвета столбцов соответствуют статусам (стандартная палитра CRM) |

### Метрики

- Количество элементов на каждом этапе
- Конверсия между этапами (процент)
- Общая сумма сделок на этапе
- Средняя длительность этапа (в днях)
- Общая конверсия воронки (первый этап к последнему)

### Фильтры

- Период (предустановленные: сегодня, неделя, месяц, квартал, год + кастомный)
- Ответственный менеджер
- Источник лида/сделки
- Любое пользовательское поле
- Тип элемента (лид/сделка)
- Конкретная воронка (при наличии нескольких)

### Уникальные особенности

1. **Два типа воронки.** Классическая воронка учитывает только фактически пройденные этапы — если сделка перескочила этап, он не отображается. Конверсионная показывает все этапы, даже пропущенные, что даёт полную картину процесса.
2. **Объединение лидов и сделок** в одной воронке — можно видеть полный путь от лида до закрытой сделки.
3. **Фильтрация по любым полям**, включая кастомные — гибкость для сложных сценариев.
4. **BI-коннектор** — возможность выгрузки данных в Google Looker Studio и Power BI для продвинутой аналитики.

### Слабые стороны

- Интерфейс перегружен, много шагов для настройки отчёта
- Визуализация базовая — нет классической трапецеидальной funnel chart
- Нет встроенного сравнения периодов (только через BI)
- Drill-down в сделки на этапе неочевиден
- Производительность страдает на больших объёмах данных

## 2. AmoCRM

### Общее описание

AmoCRM — CRM с фокусом на продажи, где воронка является центральным элементом интерфейса. Раздел «Аналитика» -> «Анализ продаж» предоставляет визуальный отчёт по воронке. Дополнительно доступен конструктор отчётов для кастомных визуализаций.

### Визуализация

| Характеристика | Описание |
|---|---|
| Тип диаграммы | Вертикальная столбчатая диаграмма с убывающей высотой |
| Альтернативные виды | Конструктор отчётов: таблица, столбчатая диаграмма, линейный график |
| Интерактивность | Клик по этапу показывает список сделок; hover показывает метрики |
| Цветовая схема | Градиент от зелёного (начало) к красному (потери) |

### Метрики

- Количество сделок на каждом этапе
- Конверсия между этапами (процент перехода)
- Общая сумма сделок
- Средний чек
- Количество новых/выигранных/проигранных сделок за период
- Эффективность менеджеров (сделки, суммы, конверсия на человека)

### Фильтры

- Период (предустановленные + кастомный диапазон)
- Воронка (при наличии нескольких)
- Ответственный менеджер (мультивыбор)
- Теги
- Бюджет сделки (диапазон)
- Статус сделки

### Уникальные особенности

1. **Цифровая воронка** — автоматизация действий при переходе между этапами (авто-задачи, авто-письма). Не относится напрямую к отчёту, но создаёт контекст для аналитики.
2. **Конструктор отчётов** — визуальный редактор для построения кастомных отчётов без навыков программирования. Выбор объекта, фильтров, группировок, способа отображения.
3. **Аналитика по менеджерам** — встроенный отчёт по эффективности каждого менеджера в привязке к воронке.
4. **Прогнозирование выручки** — на основе текущей воронки, среднего чека и исторической конверсии.

### Слабые стороны

- Нет встроенной классической funnel chart (трапецеидальная диаграмма)
- Ограниченное сравнение периодов (нет overlay)
- Экспорт данных только в Excel
- Нет Sankey-диаграммы для визуализации перетоков между этапами
- Конструктор отчётов доступен только на старших тарифах

## 3. HubSpot (Sales Hub)

### Общее описание

HubSpot предлагает мощную систему отчётов с отдельным типом «Funnel Report» в Report Builder. Доступна аналитика по сделкам (deal funnel), контактам (lifecycle funnel) и кастомным объектам. Sales Analytics Suite содержит предустановленные отчёты.

### Визуализация

| Характеристика | Описание |
|---|---|
| Тип диаграммы | Funnel bar (горизонтальная воронка), столбчатая диаграмма, таблица |
| Альтернативные виды | Переключатель Over time / Totals — динамика во времени или суммарные значения |
| Интерактивность | Клик по сегменту — drill-down в список записей |
| Цветовая схема | Градиент голубой, акцент на потерях |

### Метрики

- Количество сделок на каждом этапе
- Конверсия между этапами (cumulative и step-by-step)
- Количество «пропусков» (skips) — сделки, перескочившие этап
- Время в каждом этапе (среднее)
- Deal velocity — скорость прохождения сделок
- Average deal size
- Pipeline waterfall (изменение pipeline за период)

### Фильтры

- Период (дата создания, дата закрытия, дата входа в этап)
- Pipeline (выбор конкретного пайплайна)
- Свойства контакта (любые, включая списки)
- Свойства компании
- Свойства сделки (любые кастомные поля)
- Owner (ответственный)
- AND-логика для комбинации фильтров

### Уникальные особенности

1. **Skip tracking** — уникальная метрика: система отслеживает сделки, которые перескочили этапы, и показывает их количество отдельно. Конверсия считается только для сделок, которые фактически были на обоих этапах.
2. **Переключатель Over time / Totals** — можно смотреть воронку в динамике (по неделям/месяцам) или суммарно.
3. **Три типа данных** — отчёт по сделкам, контактам или кастомным объектам в едином интерфейсе.
4. **Sales Analytics Suite** — набор предустановленных отчётов (deal velocity, pipeline waterfall, forecast) для тех, кому не нужен кастомный отчёт.
5. **Flexible date filtering** — фильтрация по дате создания, дате закрытия ИЛИ дате входа в конкретный этап.

### Слабые стороны

- Сложность настройки — много опций, новичкам тяжело разобраться
- Funnel report доступен только на Professional и Enterprise тарифах
- Нет трапецеидальной визуализации — только горизонтальные бары
- Нет встроенного сравнения двух периодов в одном отчёте
- Ограниченная кастомизация внешнего вида графиков

## 4. Pipedrive

### Общее описание

Pipedrive — CRM, построенная вокруг пайплайна. Раздел «Insights» содержит несколько типов отчётов: Deal conversion, Deal duration, Deal performance, Activities performance. Funnel conversion — отдельный тип отчёта.

### Визуализация

| Характеристика | Описание |
|---|---|
| Тип диаграммы | Столбчатая диаграмма (column chart) |
| Альтернативные виды | Scorecard (карточки с ключевыми числами) |
| Интерактивность | Клик по столбцу — drill-down в список сделок |
| Цветовая схема | Брендовые цвета, подсветка won/lost |

### Метрики

- Конверсия между этапами (stage-to-stage)
- Конверсия open -> won / open -> lost
- Средняя длительность сделки (deal duration) по каждому этапу
- Общая длительность цикла продаж
- Deal velocity (объём и скорость прохождения)
- Started / Won / Lost count

### Фильтры

- Период (дата создания сделки)
- Pipeline (выбор пайплайна)
- Ответственный (owner)
- Любые кастомные поля сделки
- Комбинация фильтров через AND

### Уникальные особенности

1. **Раздельные отчёты** — конверсия и длительность разделены на два отчёта, каждый с фокусом на своей метрике. Это проще для восприятия.
2. **Measure-by / View-by / Segment-by** — гибкая система осей: можно менять, что измеряется (количество, сумма), по чему группируется (этап, период), чем сегментируется (owner, source).
3. **Глубокая фильтрация** — комбинация стандартных и кастомных полей в одном фильтре.
4. **Insights-дашборды** — можно собрать несколько отчётов на одном экране.

### Слабые стороны

- Нет классической funnel chart (трапецеидальная) — только столбчатая диаграмма
- Ограниченные типы визуализации (2 варианта)
- Нет сравнения периодов в одном отчёте
- Нет Sankey-диаграммы
- Insights доступен с Professional тарифа

## 5. Monday.com (Monday CRM)

### Общее описание

Monday CRM предлагает Funnel Chart Widget — виджет для дашбордов, показывающий воронку сделок. Виджет привязывается к доскам с колонкой статуса и автоматически рассчитывает конверсию. Также доступен Deal Stages widget и Leaderboard widget.

### Визуализация

| Характеристика | Описание |
|---|---|
| Тип диаграммы | Вертикальная столбчатая диаграмма (funnel bar) |
| Альтернативные виды | Deal Stages widget (горизонтальная шкала прогресса) |
| Интерактивность | Клик по столбцу — список элементов на этом этапе |
| Цветовая схема | 3 режима: по цветам статусов, одноцветный (зелёный), синий с зелёным финалом |

### Метрики

- Количество элементов на каждом этапе
- Конверсия между этапами (процент)
- Общая конверсия от первого к последнему этапу
- Количество переходов между статусами

### Фильтры

- Owner (ответственный) — выпадающий список
- Funnel start date (дата начала) — выбор периода
- Привязка к конкретной доске или нескольким доскам

### Уникальные особенности

1. **Простота настройки** — виджет добавляется на дашборд в один клик, привязывается к доске и колонке статуса. Минимальная конфигурация.
2. **No-code dashboard builder** — drag-and-drop интерфейс для сборки дашбордов из виджетов.
3. **AI-powered insights** — система подсвечивает паттерны и bottlenecks автоматически.
4. **Multi-board dashboards** — можно объединять данные с нескольких досок на одном дашборде.
5. **Leaderboard widget** — сравнение менеджеров по этапам воронки как отдельный виджет.

### Слабые стороны

- Ограниченные метрики — нет суммы сделок, среднего времени на этапе
- Нет drill-down в детали сделки из воронки
- Нет сравнения периодов
- Визуализация только столбчатая — нет funnel shape
- Фильтрация ограничена (только owner и дата)

## 6. Salesforce

### Общее описание

Salesforce предоставляет enterprise-level систему отчётности. Funnel chart — один из типов диаграмм в Report Builder. Для аналитики воронки используются комбинации отчётов: Opportunity by Stage, Pipeline Shape, Stage Duration, Conversion Rate. CRM Analytics (Tableau) для продвинутых сценариев.

### Визуализация

| Характеристика | Описание |
|---|---|
| Тип диаграммы | Funnel chart (трапецеидальная), столбчатая, таблица |
| Альтернативные виды | Pipeline shape, waterfall chart, Tableau dashboards |
| Интерактивность | Drill-down в записи, cross-filtering между виджетами |
| Цветовая схема | Полностью кастомизируемая |

### Метрики

- Количество opportunities на каждом этапе
- Сумма (amount) на каждом этапе
- Конверсия между этапами (stage-to-stage)
- Время в каждом этапе (требует кастомных полей: boolean + date для каждого этапа)
- Pipeline shape (распределение по этапам)
- Deal velocity
- Win rate
- Forecast (прогнозируемая выручка)

### Фильтры

- Период (дата создания, дата закрытия, кастомные даты)
- Stage (конкретный этап или диапазон)
- Owner / Team
- Lead Source
- Campaign
- Territory
- Любые стандартные и кастомные поля
- Dynamic filtering — фильтры работают на весь дашборд

### Уникальные особенности

1. **Настоящая funnel chart** — трапецеидальная диаграмма, где ширина секции отражает объём.
2. **Cohort analysis** — группировка конверсии по источнику лида, кампании, территории.
3. **Dynamic dashboard filters** — один фильтр применяется ко всем виджетам на дашборде.
4. **CRM Analytics (Tableau)** — продвинутая аналитика с AI-предсказаниями, Sankey-диаграммами, что-если сценариями.
5. **Pipeline Trend** — отчёт, показывающий рост или сокращение воронки во времени.
6. **Forecast categories** — прогнозирование выручки на основе вероятности закрытия по этапам.

### Слабые стороны

- Высокая сложность настройки — требует администратора Salesforce
- Трекинг времени на этапе требует кастомных полей (boolean + date) для каждого этапа
- Высокая стоимость (Enterprise тарифы, CRM Analytics — отдельная лицензия)
- Нет встроенного сравнения периодов в стандартном funnel chart
- Перегруженный интерфейс, крутая кривая обучения

## Ключевые паттерны и best practices

### 1. Визуализация

**Общий паттерн:** Большинство продуктов используют столбчатую диаграмму, а не классическую трапецеидальную funnel chart. Только Salesforce предлагает настоящую funnel shape.

**Рекомендация для Kaiten:** Реализовать трапецеидальную funnel chart как основной вид — это станет конкурентным преимуществом среди B2B инструментов. Столбчатая диаграмма и таблица — как альтернативные виды.

### 2. Метрики

**Общий паттерн:** Минимальный набор метрик, который есть у всех: количество сделок, конверсия между этапами, сумма. Продвинутые системы добавляют время на этапе и deal velocity.

**Рекомендация для Kaiten:** MVP должен включать: количество, сумму, конверсию, среднее время на этапе. Deal velocity — кандидат для второй итерации.

### 3. Фильтрация

**Общий паттерн:** Период + ответственный + источник — базовый набор. Лучшие системы поддерживают фильтрацию по любым кастомным полям.

**Рекомендация для Kaiten:** Период, ответственный, теги — в первую версию. Фильтрация по кастомным полям — во вторую. Предустановленные периоды (неделя, месяц, квартал) критически важны для быстрого доступа.

### 4. Drill-down

**Общий паттерн:** Клик по этапу -> список сделок. HubSpot и Salesforce делают это лучше всего — с сортировкой и возможностью перейти в карточку сделки.

**Рекомендация для Kaiten:** Обязательно в первой версии. Клик по этапу должен открывать боковую панель или модальное окно со списком карточек, с возможностью перейти в карточку.

### 5. Сравнение периодов

**Общий паттерн:** Ни один из исследованных продуктов не реализует полноценное сравнение периодов в встроенном отчёте воронки. HubSpot предлагает переключатель Over time, но это не прямое сравнение.

**Рекомендация для Kaiten:** Это возможность для дифференциации. Реализовать сравнение двух периодов (текущий vs. предыдущий) с delta-показателями — станет конкурентным преимуществом.

### 6. Настройка воронки

**Общий паттерн:** В большинстве CRM воронка привязана к pipeline/доске, а этапы определяются колонками. Monday.com делает это проще всего — привязка к доске и колонке статуса.

**Рекомендация для Kaiten:** Поскольку Kaiten использует доски с колонками, привязка должна быть аналогичной: выбор доски -> автоматическое определение этапов из колонок -> возможность изменить порядок и исключить ненужные.

| Быстрая настройка из доски | «Построить воронку» прямо из контекстного меню доски | Низкая | Средняя |

| Экспорт в PNG/PDF | Многие показывают только CSV/Excel, но руководителям нужен визуальный экспорт | Низкая | Средняя |

## Выводы

1. **Рынок CRM-отчётов воронки стандартизирован** — все продукты показывают одинаковый минимальный набор метрик. Дифференциация возможна через UX, визуализацию и уникальные функции.

2. **Трапецеидальная funnel chart** — неиспользованная возможность. Почти никто из основных конкурентов (кроме Salesforce enterprise) не предлагает настоящую funnel shape. Это визуально понятнее для руководителей.

3. **Сравнение периодов** — главный gap на рынке. Ни один продукт не делает это хорошо во встроенном отчёте. Реализация этой функции даст Kaiten уникальное преимущество.

4. **Простота настройки** — критический фактор. Monday.com побеждает в этом аспекте. Kaiten должен обеспечить настройку воронки за 2-3 клика: выбрал доску, увидел воронку.

5. **Drill-down обязателен** — все серьёзные CRM-системы позволяют провалиться в список сделок на этапе. Без этой функции отчёт будет восприниматься как «красивая картинка без пользы».

6. **Для MVP достаточно:** funnel chart + bar chart + таблица, 4 метрики (количество, сумма, конверсия, время на этапе), базовые фильтры (период, ответственный), drill-down в список карточек.
