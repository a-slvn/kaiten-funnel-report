# Спецификация для разработчика: Отчёт «Воронка продаж»

## Какие метрики отслеживаем

Смотрим поточно и анализируем обратную связь.

### User story

| User story | Цель / Гипотеза | Критерии готовности | Артефакты |
|---|---|---|---|

| Хочу скорректировать настройки воронки | Часть пользователей захочет уточнить автоконфигурацию | Панель: включение/исключение досок и колонок, перетаскивание, выбор «выиграно/проиграно», поле суммы; сохранение | Панель настроек |

## Use Cases

| Use Case | Описание |
|---|---|
| Открыть отчёт (первый раз) | Воронка строится автоматически по всем доскам пространства. Если есть неоднозначности — показываются баннеры-предупреждения. |
| Открыть отчёт (повторно) | Загружается ранее сохранённая конфигурация. Предупреждения автоконфигурации не показываются. |
| Открыть отчёт с автоподбором метрики | Если суммы доступны, отчёт сразу открывается в денежном режиме. Если суммы недоступны, отчёт открывается в count-fallback с alert. |
| Применить фильтр | Изменение периода, ответственного, тегов, суммы. Воронка перестраивается. Обновляется бейдж фильтров. |
| Drill-down в этап | Клик по этапу воронки. Открывается боковая панель со списком сделок на этом этапе. |
| Настроить воронку | Открытие панели настроек. Изменение конфигурации. Сохранение. Перестроение воронки. |
| Сравнить периоды | Включение режима сравнения. Выбор второго периода. Отображение двух воронок рядом с разницей. |
| Экспорт данных | Скачивание данных в CSV, Excel или PNG. Учитываются текущие фильтры. |

## Ручная настройка и пересчёт

После нажатия `Применить` в модалке настройки график, таблица, KPI и drill-down должны использовать один и тот же список включённых колонок.

### Правила для FE

1. В график и таблицу попадают все включённые колонки из модалки, в том же порядке: сначала порядок досок, затем порядок колонок внутри доски.
2. Колонка с ролью `Этап воронки` отображается как обычный этап.
3. Колонка с ролью `Успешный результат` отображается как финальная колонка успеха.
4. Колонка с ролью `Проиграно` отображается как финальная колонка потерь.
5. Если колонка выключена в модалке, она сразу исключается из визуализации и из расчёта.

### Правила пересчёта

1. Сделка в статусе `in_progress` учитывается только если её текущая колонка включена в модалке.
2. Сделка в статусе `won` переносится в первую включённую колонку роли `Успешный результат` на той же доске.
3. Сделка в статусе `lost` переносится в первую включённую колонку роли `Проиграно` на той же доске.
4. Если для `won` или `lost` на доске нет включённой колонки соответствующей роли, такие сделки временно исключаются из воронки до тех пор, пока пользователь не включит подходящую колонку.
5. После переноса сделок пересчитываются:
   - набор колонок на графике и в таблице;
   - количество сделок на каждой колонке;
   - конверсия в следующую колонку;
   - итоговые KPI;
   - содержимое drill-down по клику на колонку.

### Ограничение текущего mock-режима

В mock-данных закрытая сделка хранит только исходную колонку этапа и итоговый статус `won/lost`, без точного ID финальной done-колонки. Поэтому при нескольких включённых колонках одной и той же роли на одной доске все сделки этого исхода попадают в первую включённую колонку этой роли.

## Открыть отчёт (первый раз)

### Требования для FE

При первом открытии отчёта пользователь сразу видит воронку — без экрана настройки.

Сначала система проверяет, указан ли на досках параметр «Сумма по карточкам». Если да — отчёт сразу строится по сумме. Если нет — отчёт строится только по количеству карточек.

Если числовое поле суммы не найдено:
- в шапке графика не показывается выбор `Сумма`;
- в модалке `Настроить воронку` не показывается блок выбора поля суммы;
- над графиком показывается alert о том, что воронка построена по количеству карточек;
- в alert есть ссылка на статью, как добавить параметр суммы в Kaiten: `https://faq-ru.kaiten.site/99d5507a-4522-4641-927b-4d99954be51e`.

Если поле суммы не определено или не может быть использовано как единая метрика пространства:
- в модалке `Настроить воронку` не показывается блок выбора поля суммы;
- alert объясняет, что сумму нужно настроить на досках;
- кнопка действия в alert ведёт в документацию Kaiten, а не в настройку отчёта.

### Требования для BE

Если для пространства нет сохранённой конфигурации — бекенд автоматически определяет структуру воронки:
- Колонки выстраивает в порядке слева-направо
- Последнюю колонку с типом «Готово» определяет как «выиграно». Колонка, которая стоит после «выиграно», определяется как «проиграно»
- Ищет общее числовое поле на всех досках для расчёта суммы

Если обнаружены неоднозначности (несколько завершающих колонок, разные поля суммы и т.д.) — в ответ добавляются предупреждения.

## Автовыбор метрики

### Требования для FE

Фронтенд не показывает пользователю кнопки переключения `Количество / Сумма` в блоке графика.

Если поле суммы определено, отчёт по умолчанию строится в режиме `Сумма`:

- на этапах показываются денежные значения;
- ось графика считается по сумме;
- KPI используют денежную метрику там, где она применима.

Если поле суммы не определено или не может быть использовано как единая метрика пространства, отчёт автоматически падает в режим `Количество`:

- на графике показываются только количества сделок;
- ось графика считается по количеству;
- причина fallback объясняется через alert над графиком.

Конверсия между этапами всегда считается по количеству сделок — не зависит от выбранного режима.

Средний чек показывается только в режиме «Сумма» (в тултипе при наведении на этап).

### Требования для BE

Бекенд возвращает и количество, и сумму в одном ответе. Фронтенд выбирает, что показать по умолчанию, без дополнительного запроса.

Если поле суммы не определено — бекенд в ответе указывает, что доступен только режим `Количество`, и добавляет соответствующее предупреждение.

## Фильтры

### Период

#### Требования для FE

Выпадающий список с предустановленными значениями:

| Значение | Описание |
|---|---|
| Сегодня | С начала текущего дня |
| Последние 7 дней | 7 календарных дней до сегодня |
| Текущий месяц | С 1-го числа текущего месяца (по умолчанию) |
| Предыдущий месяц | Полный предыдущий месяц |
| Текущий квартал | С начала текущего квартала |
| Текущий год | С 1 января текущего года |
| Кастомный | Произвольный диапазон — появляется выбор дат |

При выборе «Кастомный» — появляется выбор начальной и конечной даты. Начальная дата не может быть позже конечной.

#### Требования для BE

В отчёт попадают сделки по когортному принципу: учитывается дата первого входа сделки на любой этап воронки. Если сделка впервые вошла в воронку в выбранный период — она попадает в отчёт, и все её дальнейшие перемещения тоже учитываются (даже если они произошли позже).

Максимальный период — 366 дней.

### Ответственный

#### Требования для FE

Мультивыбор с поиском по имени. Список формируется из участников всех досок пространства. Есть кнопки «Выбрать всех» и «Сбросить». По умолчанию ничего не выбрано — показываются все сделки.

При выборе нескольких менеджеров — показываются сделки любого из выбранных (логика «ИЛИ»).

#### Требования для BE

Фильтрация по полю «Ответственный» карточки. Все переданные ID должны быть валидными участниками досок пространства.

### Теги

#### Требования для FE

Мультивыбор с поиском. Список формируется из тегов карточек на досках пространства. Логика «ИЛИ»: карточка проходит фильтр, если содержит хотя бы один из выбранных тегов.

#### Требования для BE

Фильтрация по тегам карточек. Карточка попадает в отчёт, если у неё есть хотя бы один из указанных тегов.

### Сумма сделки (от / до)

#### Требования для FE

Два числовых поля: «от» и «до». Можно заполнить одно из них. Нечисловые символы не вводятся. Значение «до» не может быть меньше «от» — при нарушении поле подсвечивается красным.

Поля заблокированы, если поле суммы не определено (режим только «Количество»).

#### Требования для BE

Фильтрация по значению поля суммы карточки. Если заданы оба значения — выбираются сделки в диапазоне. Если одно — фильтрация только по нижней или верхней границе.

### Панель фильтров — общее поведение

| Элемент | Описание |
|---|---|
| Индикатор | Бейдж с количеством активных фильтров (кроме периода) |
| Кнопка «Сбросить» | Появляется при наличии хотя бы одного активного фильтра (кроме периода). Сбрасывает всё, период остаётся |
| Применение | Фильтры применяются мгновенно при изменении — без кнопки «Применить» |
| Сохранение | Фильтры НЕ сохраняются между сессиями (только конфигурация воронки сохраняется) |

## Визуализация воронки

### Диаграмма воронки (основной вид)

#### Требования для FE

Вертикальная трапецеидальная диаграмма (сверху вниз). Ширина каждого этапа пропорциональна значению.

На каждом этапе отображается:
- Название этапа — слева от секции
- Количество сделок — внутри секции (например, «147 сделок»)
- Сумма — внутри секции, только в режиме «Сумма»
- Конверсия от предыдущего этапа — между секциями (например, «72%»)
- Среднее время — в тултипе при наведении

При наведении на этап — подсветка секции и тултип с расширенными метриками. При клике — открытие drill-down.

Цветовая индикация конверсии:
- 70% и выше — зелёный
- От 40% до 69% — оранжевый (амбер)
- Ниже 40% — красный

### Таблица (альтернативный вид)

#### Требования для FE

Столбцы таблицы:
- Этап (название)
- Количество сделок
- Сумма (прочерк если недоступна)
- Средний чек (прочерк если недоступен)
- Конверсия от предыдущего (прочерк для первого этапа)
- Конверсия общая (от первого этапа)
- Среднее время на этапе (в днях)

Сортировка по любому столбцу.

### Переключатель видов

Три варианта отображения: воронка, столбчатая диаграмма, таблица. Расположение — правый верхний угол области визуализации. Переключение без перезагрузки данных. Выбранный вид сохраняется в localStorage.

### Пустое состояние

Если по текущим фильтрам нет данных:
- Иконка пустой воронки (серая, 64px)
- Заголовок: «Нет данных для отображения»
- Подпись: «Попробуйте изменить фильтры или выбрать другой период»
- Кнопка «Сбросить фильтры» (если фильтры активны)

## Алерты автоконфигурации

### Требования для FE

Алерты информируют пользователя о неоднозначностях автоматического определения воронки. Воронка при этом строится и показывается — алерты её не блокируют (кроме ошибок).

Каталог алертов:

| Тип | Сообщение | Действие | Когда показывается |
|-----|-----------|----------|--------------------|
| Ошибка | На пространстве нет досок. Создайте доску. | — | 0 досок в пространстве |
| Ошибка | Все колонки имеют тип «Готово». Добавьте колонки с другими типами. | — | Все колонки завершающие |
| Предупреждение | Нет колонок типа «Готово». Последняя колонка использована как «Выигран». | Перейти в настройки | 0 завершающих колонок |
| Информация | Найдена одна завершающая колонка — «Выигран». Укажите «Проигран» в настройках. | Настроить | 1 завершающая колонка |
| Предупреждение | Найдено несколько завершающих колонок. Последняя = «Выигран», остальные = «Проигран». | Настроить | 3 и более завершающих |
| Информация | На досках нет числового поля для суммы сделки. Воронка построена по количеству карточек. Настройте поле суммы на досках, чтобы видеть суммы. | Как настроить сумму | Нет числовых полей |
| Предупреждение | На разных досках разные числовые поля. Воронка по количеству, потому что суммы несопоставимы. Настройте одно общее поле суммы на досках. | Как настроить сумму | Разные поля на досках |
| Предупреждение | Не на всех досках есть числовое поле. Воронка по количеству. Настройте одинаковое поле суммы на всех досках. | Как настроить сумму | Поле есть не везде |
| Информация | Найдено несколько общих числовых полей. Воронка по количеству, пока на досках не останется одно общее поле суммы. | Как настроить сумму | Больше одного подходящего поля |

Правила отображения:
- **Ошибки** блокируют график — вместо воронки показывается сообщение
- **Предупреждения** — баннер над графиком, график виден
- **Информация** — сворачиваемый блок над графиком
- Одновременно не более 2 алертов. Приоритет: ошибка > предупреждение > информация. Остальные — по ссылке «ещё»
- Алерт можно закрыть крестиком. В рамках сессии не появляется снова. При перезагрузке — появляется
- Если пользователь сохранил свою конфигурацию — алерты автоконфигурации не показываются

Дизайн: полноширинный баннер (компонент Alert). Иконка слева, текст в центре, кнопка действия справа, крестик закрытия. Фон: голубой (информация), жёлтый (предупреждение), красный (ошибка).

Для всех amount-related alert (`NO_AMOUNT_FIELD`, `DIFFERENT_AMOUNT_FIELDS`, `PARTIAL_AMOUNT_FIELDS`, `MULTIPLE_AMOUNT_FIELDS`) кнопка действия открывает статью Kaiten FAQ о настройке числового поля суммы: `https://faq-ru.kaiten.site/99d5507a-4522-4641-927b-4d99954be51e`.

### Требования для BE

Алерты формируются на бекенде при выполнении автоконфигурации и возвращаются вместе с данными отчёта. Если конфигурация сохранена пользователем вручную — алерты не генерируются.

## Детализация этапа (Drill-down)

### Требования для FE

При клике на этап воронки справа выезжает боковая панель шириной 480px.

Заголовок панели: название этапа + количество сделок (например, «Переговоры (47 сделок)»).

Закрытие: кнопка X, клик по затемнённому фону, нажатие Esc.

Для каждой сделки в списке показывается:
- Название карточки (кликабельная ссылка — открывает карточку в Kaiten)
- Ответственный (маленький аватар + имя)
- Сумма (например, «125 000 руб.»; прочерк если сумма недоступна)
- Время на этапе (например, «12 дней»)
- Дата создания (например, «15 янв 2026»)

Сортировка:
- По умолчанию — по времени на этапе (убывание, самые зависшие сверху)
- Доступны: по времени, по сумме, по дате создания, по ответственному

Пагинация: 25 сделок на странице. Кнопки «Назад» / «Вперёд» + номера страниц. Подпись «Показано 1–25 из 47».

Кнопка «Экспорт списка» в заголовке — скачивает CSV со всеми сделками этапа (не только текущую страницу).

### Требования для BE

Бекенд возвращает список сделок для конкретного этапа с учётом текущих фильтров. Поддерживает пагинацию (по 25 элементов, максимум 100) и сортировку (по времени на этапе, сумме, дате создания, имени ответственного).

Для экспорта — отдельный запрос, который возвращает все сделки этапа без пагинации.

## Настройка воронки

### Требования для FE

Кнопка «Настройки» (иконка шестерёнки) открывает боковую панель или диалог с текущей конфигурацией.

Элементы настройки:

| Поле | Описание | Ограничения |
|---|---|---|
| Доски | Чекбоксы для каждой доски пространства | Минимум 1 должна быть включена |
| Этапы (колонки) | Чекбоксы + перетаскивание для порядка | Минимум 2 должны быть включены |
| Колонка «Выиграно» | Выпадающий список из завершающих колонок | Обязательно выбрать хотя бы одну |
| Колонки «Проиграно» | Мультивыбор из завершающих колонок | Необязательно |
| Поле суммы | Выпадающий список числовых полей + «Не выбрано» | — |

Кнопки:
- **«Сохранить»** — сохраняет конфигурацию, перестраивает воронку. Заблокирована при невалидном состоянии (меньше 2 этапов, нет включённых досок)
- **«Сбросить к автоопределению»** — удаляет сохранённый конфиг, возвращает автоматическое определение
- **«Отмена»** — закрытие без сохранения

### Требования для BE

Конфигурация сохраняется на уровне пространства. Если для пространства уже есть конфиг — он обновляется (не создаётся новый).

При удалении конфига — бекенд снова использует автоопределение при следующем запросе отчёта.

Валидации: все доски принадлежат пространству; минимум 2 этапа; колонки «выиграно» — завершающего типа; поле суммы — числовое.

## Сравнение периодов

### Требования для FE

Кнопка «Сравнить периоды» рядом с фильтром периода. При нажатии появляется второй выбор дат. По умолчанию — «Предыдущий период» той же длительности (например, если текущий — январь, то сравнение — декабрь).

Отображение: две воронки рядом (основной период слева, сравнение справа). Между ними — разница на каждом этапе.

Цвет разницы: зелёный (рост), красный (падение), серый (без изменений).

Формат разницы:
- Количество: «+12» или «-5»
- Сумма: «+1 250 000 руб.»
- Конверсия: «+3.2 п.п.» или «-1.5 п.п.»

Выключение: кнопка X рядом со вторым периодом — возврат к одиночному виду.

### Требования для BE

Бекенд принимает даты второго периода и возвращает данные для обоих периодов + рассчитанную разницу. Если второй период не указан — поле сравнения в ответе пустое.

## Экспорт

### Требования для FE

Кнопка «Экспорт» в правом верхнем углу → выпадающее меню:
- **CSV** — таблица с этапами: название, количество, сумма, средний чек, конверсии, среднее время
- **Excel** — то же + заголовок с информацией о фильтрах. Если включено сравнение — добавляются столбцы разницы
- **PNG** — текущая диаграмма с заголовком и легендой (белый фон, 1920x1080)

Все форматы учитывают текущие фильтры.

### Требования для BE

Отдельные запросы для экспорта данных (CSV/Excel) и экспорта drill-down (CSV со всеми сделками этапа без пагинации). В заголовке файла указываются пространство, период и применённые фильтры.

## Секция «По сотрудникам»

### Требования для FE

Расположена под основной воронкой. Группировка по ответственному менеджеру. Каждый менеджер — раскрывающийся блок (expand/collapse).

Для каждого менеджера: количество сделок по этапам, конверсия, средний чек, среднее время.

Сортировка по общему количеству сделок (убывание). Менеджеры без сделок в текущем периоде не отображаются.

### Требования для BE

Бекенд дополнительно группирует данные по ответственному. Возвращает массив: для каждого менеджера — его данные и метрики по этапам.

## Validations and Errors

### Frontend

| Элемент | Валидация | Вывод ошибки на FE | Проверки |
|---|---|---|---|
| Пространство | Должно быть определено из контекста навигации | «Выберите пространство для построения воронки» | Обязательное поле |
| Доски (настройки) | Минимум 1 включённая доска | Кнопка «Сохранить» заблокирована; текст «Включите минимум 1 доску» | Обязательное поле |
| Этапы (настройки) | Минимум 2 включённых | Кнопка «Сохранить» заблокирована; текст «Выберите минимум 2 этапа» | Обязательное поле |
| Начальная дата | Не позже конечной | Подсветка красным, тултип «Начальная дата не может быть позже конечной» | Обязательное поле |
| Сумма «от» | Не отрицательная | Не принимается отрицательное значение | Числовое поле |
| Сумма «до» | Не меньше «от» | Подсветка красным при нарушении | Числовое поле |
| Сумма «от» / «до» | Только числа | Нечисловые символы не вводятся | Числовое поле |
| Сумма «от» / «до» | Доступность | Заблокированы, если поле суммы не определено | Условная доступность |

### Backend

| Поле | Правило / Причина появления ошибки | Возможная ошибка | Решается на |
|---|---|---|---|
| Пространство | Не существует или нет доступа | Ошибка 403 — «Нет доступа к пространству» | BE |
| Пространство | 0 неархивных досок | Ошибка 400 — «В пространстве нет досок» | BE |
| Доски | Доска не принадлежит пространству | Ошибка 400 — «Доска не принадлежит пространству» | BE |
| Этапы | Менее 2 валидных колонок | Ошибка 400 — «Необходимо минимум 2 этапа» | BE |
| Начальная дата | Некорректная дата | Ошибка 400 — «Некорректная начальная дата» | BE |
| Конечная дата | Раньше начальной | Ошибка 400 — «Конечная дата не может быть раньше начальной» | BE |
| Период | Больше 366 дней | Ошибка 400 — «Максимальный период 366 дней» | BE |
| Поле суммы | Не существует, не числовое, не принадлежит доскам | Ошибка 400 — «Поле суммы не найдено или неверный тип» | BE |
| Ответственные | Несуществующий участник | Ошибка 400 — «Указан несуществующий ответственный» | BE |
| Конфигурация | Структура пространства изменилась с момента сохранения | Ошибка 409 — предложение обновить конфигурацию | FE (перезапрос) |

## QA

[Полный набор тест-кейсов](../qa/test-cases-funnel.md)

### Ключевые проверки

| Проверка | Ожидание |
|---|---|
| Первый заход — воронка строится сразу | График появляется быстро; нет экрана настройки |
| Предупреждения не блокируют график | Воронка видна под баннерами |
| Ошибки блокируют график | Вместо графика — сообщение |
| Фильтры работают мгновенно | Данные обновляются при каждом изменении |
| Drill-down соответствует цифрам | Количество карточек в панели = число на этапе |
| Автовыбор суммы или count-fallback | Если сумма доступна, отчёт сразу открыт в денежном режиме; если суммы нет, отчёт сразу открыт в количестве и показывает alert |
| Вариант «Сумма» скрыт | Если нет общего числового поля — пользователь остаётся только в режиме «Количество», причина объясняется alert-баннером |
| Настройки сохраняются | При повторном входе — сохранённый конфиг |
| Сброс к автоопределению | Конфиг удаляется → воронка строится заново автоматически |
| Пустое состояние | Если 0 сделок — иконка, текст, кнопка «Сбросить фильтры» |
