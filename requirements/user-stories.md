# User Stories: Отчёт «Воронка продаж»

## Роли

| Роль | Описание | Ключевые потребности |
|---|---|---|
| Руководитель отдела продаж | Управляет командой менеджеров, отвечает за выполнение плана продаж | Видеть общую картину, находить узкие места, контролировать команду |
| Менеджер по продажам | Работает с конкретными сделками, ведёт клиентов по воронке | Видеть свои показатели, понимать где теряются сделки |
| Собственник бизнеса / CEO | Принимает стратегические решения, контролирует бизнес на верхнем уровне | Быстро оценить здоровье воронки, сравнить периоды, спрогнозировать выручку |
| Аналитик | Готовит отчёты для руководства, ищет закономерности в данных | Гибкие фильтры, экспорт данных, детализация по сегментам |

## Просмотр общей воронки

| N | Роль | Задача (Как...) | Цель (Чтобы...) | Критерии готовности |
|---|---|---|---|---|
| 1.1 | Руководитель отдела продаж | Как руководитель, хочу открыть отчёт «Воронка продаж» и сразу увидеть текущее состояние воронки по всему пространству | Чтобы за 10 секунд понять, на каком этапе находятся сделки и где потенциальные проблемы | Отчёт загружается за <3 сек; воронка строится автоматически (Best Guess) по всем доскам пространства; отображается funnel chart с количеством сделок и конверсией на каждом этапе; по умолчанию показываются данные за текущий месяц; никакой предварительной настройки не требуется |
| 1.2 | Собственник / CEO | Как собственник, хочу видеть общую сумму сделок на каждом этапе воронки | Чтобы оценить потенциальную выручку и понять, достаточно ли сделок в работе для выполнения плана | На каждом этапе отображается сумма сделок (из кастомного поля «Сумма»); видна общая сумма по всей воронке |
| 1.3 | Менеджер по продажам | Как менеджер, хочу видеть воронку только по своим сделкам | Чтобы понять, на каком этапе застряли мои сделки и куда направить усилия | Фильтр «Ответственный» позволяет выбрать себя; воронка перестраивается и показывает только мои сделки |
| 1.4 | Руководитель отдела продаж | Как руководитель, хочу переключаться между видами визуализации (воронка, столбцы, таблица) | Чтобы выбрать наиболее удобный формат для текущей задачи — визуальный обзор или точные цифры | Доступны 3 вида: funnel chart, bar chart, таблица; переключение без перезагрузки данных; выбранный вид сохраняется между сессиями |

## Анализ конверсии по этапам

| N | Роль | Задача (Как...) | Цель (Чтобы...) | Критерии готовности |
|---|---|---|---|---|
| 2.1 | Руководитель отдела продаж | Как руководитель, хочу видеть процент конверсии между каждой парой соседних этапов | Чтобы быстро определить, на каком переходе теряется больше всего сделок | Между этапами отображается процент конверсии (N этапа / N предыдущего этапа * 100%); визуально подсвечиваются переходы с конверсией ниже порогового значения |
| 2.2 | Аналитик | Как аналитик, хочу видеть общую конверсию воронки (от первого этапа до последнего) | Чтобы оценить эффективность воронки целиком и отслеживать динамику | Отображается общая конверсия: количество сделок на последнем этапе / количество на первом * 100% |
| 2.3 | Руководитель отдела продаж | Как руководитель, хочу видеть среднее время нахождения сделки на каждом этапе | Чтобы выявить этапы, где сделки «зависают» дольше нормы, и принять меры | Для каждого этапа отображается среднее время в днях (рассчитывается по истории перемещений карточек); подсвечиваются этапы с аномально долгим временем |
| 2.4 | Собственник / CEO | Как собственник, хочу видеть средний чек на каждом этапе воронки | Чтобы понять, отсеиваются ли крупные сделки на ранних этапах или мелкие | На каждом этапе отображается средний чек = сумма сделок на этапе / количество сделок на этапе |

| 3.8 | Менеджер по продажам | Как менеджер, хочу видеть количество активных фильтров | Чтобы понимать, что данные отфильтрованы и воронка показывает не полную картину | Рядом с панелью фильтров отображается бейдж с количеством активных фильтров; при наведении — список применённых фильтров |

## Автопостроение воронки (Best Guess)

> **Контекст изменения:** По результатам ревью CPO (февраль 2026) секция полностью переработана. Подход "обязательная настройка перед показом графика" заменён на "Best Guess -- график строится сразу". Подробности: `requirements/cpo-feedback-changelog.md`, пункты P-001, P-002, P-003, F-001.

| N | Роль | Задача (Как...) | Цель (Чтобы...) | Критерии готовности |
|---|---|---|---|---|
| 6.1 | Руководитель отдела продаж | Как руководитель, хочу открыть воронку и сразу увидеть график без какой-либо предварительной настройки | Чтобы не тратить время на конфигурацию и сразу получить аналитику | При первом открытии отчёта воронка строится автоматически по алгоритму Best Guess: собираются все доски пространства, колонки выстраиваются в порядке sort_order, won/lost определяются по `column_type = 'done'`; пользователь видит график за <3 сек; экран настройки не показывается |
| 6.2 | Руководитель отдела продаж | Как руководитель, хочу видеть единую воронку по всему пространству, а не отдельную по каждой доске | Чтобы видеть полную картину прохождения сделок через весь процесс, даже если он разделён на несколько досок | Воронка строится по всем доскам пространства как единый поток; колонки всех досок объединяются в одну последовательность; в настройках видно, с каких досок собраны этапы |
| 6.3 | Руководитель отдела продаж | Как руководитель, хочу видеть воронку по количеству сделок, даже если поле суммы не настроено | Чтобы всегда иметь хотя бы базовую аналитику без зависимости от настройки числовых полей | По умолчанию воронка строится по количеству карточек (режим count); если числовое поле суммы определено -- доступен переключатель «Количество / Сумма»; если поле суммы не определено -- переключатель «Сумма» неактивен, воронка работает только в режиме count |

## Алерт автоконфигурации

| N | Роль | Задача (Как...) | Цель (Чтобы...) | Критерии готовности |
|---|---|---|---|---|
| 6.4 | Руководитель отдела продаж | Как руководитель, хочу видеть подсказку, если система не может точно определить выигранные/проигранные колонки | Чтобы понимать, что автоконфигурация может быть неточной, и скорректировать её при необходимости | Если Best Guess обнаруживает неоднозначность (несколько done-колонок, непонятно какая won, а какая lost) -- над графиком показывается алерт-баннер; алерт не блокирует график -- воронка построена и отображается; алерт содержит текст проблемы и кнопку «Настроить»; алерт можно скрыть (dismiss); после ручной настройки алерт не показывается |
| 6.5 | Руководитель отдела продаж | Как руководитель, хочу видеть предупреждение, если на досках пространства настроены разные поля суммы | Чтобы понимать, почему воронка показывает количество, а не суммы | Если обнаружен конфликт полей суммы (разные поля на разных досках) -- показывается алерт; воронка автоматически переключается в режим count; алерт объясняет причину и предлагает выбрать единое поле |

| 6.10 | Руководитель отдела продаж | Как руководитель, хочу сохранить скорректированные настройки для повторного использования | Чтобы при следующем входе видеть воронку с моими настройками, а не с Best Guess | Настройки (включённые доски, порядок этапов, исключённые колонки, поле суммы, won/lost колонки) сохраняются автоматически при нажатии «Применить»; при следующем входе применяется сохранённая конфигурация; есть кнопка «Сбросить к автоопределению» для возврата к Best Guess |
